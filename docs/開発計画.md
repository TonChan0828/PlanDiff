# 開発計画 — PlanDiff

**最終更新**: 2026-07-14(P5-1=定期予定 実装・自動テスト・手動疎通完了。PRレビュー待ち)
**運用ルール**:
- 実装着手時はこの表の「次に着手」から選び、`.claude/skills/spec-driven-dev` のワークフロー(仕様書作成 → ユーザー承認 → シナリオ展開 → TDD)で進める
- 着手時に「状態」を更新し、仕様書を作成したら「仕様書」列にパスを記入する
- 優先度・スコープの変更はユーザーの承認を得てからこのファイルに反映する

**優先度の定義**:
- **P0**: 外部リードタイムがある/全機能の前提。最優先で着手
- **P1**: MVPのコア体験。P0完了後に順次
- **P2**: MVPに含むが、コア体験の後でよいもの
- **P3**: MVP公開準備(ベータ投入に必要)
- **v2**: MVPスコープ外。**先回りで実装しない**(CLAUDE.md参照)

**状態**: `未着手` / `仕様書作成中` / `承認待ち` / `実装中` / `完了`
