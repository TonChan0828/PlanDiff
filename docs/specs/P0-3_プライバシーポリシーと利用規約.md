## 目的

Google OAuth審査申請(P0-5)に必須の公開ページを最短で用意する。
LPの磨き込みは後回し(P4-3)とし、審査要件を満たす法的ページ2枚+最小の導線のみを作る。

## 仕様

### ルーティング・構成

| ルート | ファイル | 内容 |
|---|---|---|
| `/privacy` | `app/(marketing)/privacy/page.tsx` | プライバシーポリシー |
| `/terms` | `app/(marketing)/terms/page.tsx` | 利用規約 |
| `/`(既存) | `app/page.tsx` | フッターに `/privacy` `/terms` へのリンクを追加(最小限の変更のみ) |

- すべて Server Component(`"use client"` なし)。静的生成される本文のみのページ
- `(marketing)` グループに `layout.tsx` を新設し、共通フッター(プライバシーポリシー・利用規約リンク、コピーライト)を持たせる
- 各ページに `metadata`(title: 「プライバシーポリシー | PlanDiff」等)を設定
- 本文は日本語。UIテキストはページコンポーネント内に定数として1箇所に寄せる(CLAUDE.mdのi18n方針)
- スタイルはTailwindで最小限(見出し・本文・余白)。375px幅で横スクロールが発生しないこと

### プライバシーポリシーの必須記載事項(OAuth審査要件)

以下のセクションを含める。**具体的な条文は実装時に作成し、コミット前にユーザーがレビューする**:

1. **事業者情報**: 運営者名・連絡先(→ 承認時の確認事項1)
2. **取得する情報**:
   - Googleアカウント情報(メールアドレス、氏名、プロフィール画像)
   - Googleカレンダーのデータ(プライマリカレンダーの予定タイトル・開始/終了日時。読み取り専用スコープ `calendar.events.readonly`)
   - ユーザーがアプリ内で記録した実績データ
3. **利用目的**: 予定と実績の表示・ギャップ分析という本機能の提供のみ。広告目的・第三者提供には使わない
4. **Google Limited Use への準拠宣言**(審査必須の文言):
   - 「PlanDiff による Google API から受領した情報の使用は、Limited Use の要件を含む [Google API Services User Data Policy](https://developers.google.com/terms/api-services-user-data-policy) に準拠します」
   - 英文の準拠宣言も併記する(審査官が確認するため)
5. **データの保存**: Supabase(PostgreSQL)に保存。カレンダーデータは表示用キャッシュとして保存。認可トークンはサーバーのみがアクセス可能な形で保管
6. **第三者提供**: 法令に基づく場合を除き行わない
7. **データの削除**: アカウント削除・データ全削除機能を設定画面から提供(提供までの間は連絡先への依頼で削除対応する旨を記載)。Google アカウント権限ページ(myaccount.google.com/permissions)からの連携解除方法も案内
8. **Cookie**: 認証セッション管理のみに使用。広告・トラッキング目的のCookieは使用しない
9. **改定**: 改定時は本ページで告知
10. **制定日・お問い合わせ先**

### 利用規約の記載事項(最小版)

1. 適用(本規約はPlanDiffの利用に適用される)
2. サービス内容(予定と実績のギャップ可視化。ベータ提供中であること)
3. アカウント(Googleアカウントでの登録、自己管理責任)
4. 禁止事項(不正アクセス、リバースエンジニアリング、法令違反等)
5. 知的財産権
6. 免責(現状有姿での提供、データ消失・損害への責任制限、ベータ期間中のサービス変更・停止)
7. サービスの変更・終了
8. 規約の変更
9. 準拠法・管轄(日本法・運営者所在地を管轄する裁判所 → 承認時の確認事項1に依存)

## テストシナリオ

境界値シナリオなし(入力・状態を持たない静的ページのため該当なし)。
DB・API連携を持たないため、結合レベルは「ページ間のリンク導線」と「ビルドでのルート生成」で担保する。→ この例外の妥当性も承認対象。

- S1 [単体] 正常系: Given プライバシーポリシーページ When レンダリング Then 見出し「プライバシーポリシー」と必須セクション(取得する情報 / Limited Use準拠宣言 / データの削除 / お問い合わせ)が表示される
- S2 [単体] 正常系: Given 利用規約ページ When レンダリング Then 見出し「利用規約」と必須セクション(免責 / 禁止事項 / 準拠法)が表示される
- S3 [単体] 正常系: Given プライバシーポリシーページ When レンダリング Then Google API Services User Data Policy へのリンク(href=https://developers.google.com/terms/api-services-user-data-policy)が存在する
- S4 [結合] 正常系: Given (marketing)レイアウトを含むトップページ When レンダリング Then フッターに `/privacy` と `/terms` へのリンクが存在する(レイアウト×ページのコンポーネント連携)
- S5 [結合] 異常系: Given 本文定数に必須文言が欠けた状態(Limited Use宣言なし) When S1/S3相当の検証を実行 Then テストが失敗する(= 必須文言の削除・変更をテストが検知できることをレビューで確認する。実装ではRedフェーズで確認)
- S6 [結合] 正常系: Given `npm run build` When 完了 Then `/privacy` `/terms` が静的ルートとして出力される(ビルド出力で確認。自動テストではなくDoDの手動確認項目)
