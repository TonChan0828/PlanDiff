# 仕様書: P3-1 オーバーレイ表示

- ステータス: 承認済み(2026-07-08)
- 関連: docs/要件定義書.md FR-06 / docs/指示資料(該当資料なし。README以外未配置)
- 前提の現状把握:
  - 予定=左レーン(55%)・薄い塗り(`bg-sky-100/80`)、実績=右レーン(45%)・濃い塗り(`bg-sky-600/90`)への分離は P2-1/P2-2 で実装済み
  - 未実装なのは FR-06 の残り2点: **(a) 予定と紐づく実績/紐づかない実績の色分け**、**(b) 開始遅延・超過(ズレ)の視覚化**
  - `TimeEntryItem`(確定済み実績の型)に `googleEventId` が含まれておらず、紐づき判定ができない。`RunningEntry` にはすでに存在する

## 目的

カレンダー画面で「予定と実績がどれだけズレたか」を一目で把握できるようにする。特に (1) 実績が予定に紐づいているか(紐づく=同系色/フリーや割り込み=別色)、(2) 紐づく実績の開始遅延・超過時間、を視覚的に伝える。

## 仕様

### 1. データ拡張(紐づき判定に必要)

- `lib/timer/types.ts`: `TimeEntryItem` に `googleEventId: string | null` を追加
- `lib/timer/entries.ts`: `fetchTimeEntries` の `select` に `google_event_id` を追加し、`googleEventId` としてマッピングする(`fetchRunningEntry` と同じパターン)
- `lib/timer/blocks.ts`: `actualBlockInputs` が確定済みエントリからも `googleEventId` を引き継ぐ(実行中エントリは既存のまま)

### 2. 紐づき判定とズレ計算(新規: `lib/timer/gap.ts`)

その日の予定ブロック(`CalendarViewEvent[]`、`googleEventId` を持つ)と実績入力(`ActualBlockInput`)を受け取り、各実績ごとに以下を返す純粋関数 `computeActualGap` を新設する。

```ts
interface ActualGapInfo {
  linked: boolean;           // googleEventIdが日内の予定と一致するか
  startDelayMinutes: number; // 実績開始 − 対応予定開始(分、0未満は0に丸める)
  overrunMinutes: number;    // 実績終了 − 対応予定終了(分、0未満は0に丸める)
}
```

判定ルール:

- `googleEventId` が `null`、または同日の予定に一致するものがない(予定削除等)場合 → `linked: false`、delay/overrunは常に0
- 同じ `googleEventId` を持つ実績が複数ある場合(中断・再開)、**時系列で最初の実績にのみ `startDelayMinutes` を計算し、最後の実績にのみ `overrunMinutes` を計算する**(中間の実績は両方0)。理由: 「予定を開始するのが遅れたか」「予定より長引いたか」は個々の再開セグメントではなく作業全体で判断すべきため
- 実績開始が予定開始より早い(早期開始)、実績終了が予定終了より早い(早期終了)は **本仕様のスコープ外**(0として扱い、装飾なし)。要件定義書 FR-06 が明記するズレは「開始遅延・超過時間」のみのため。早期終了・早期開始の可視化はP3-2(ギャップサマリー)以降で扱うか別途判断する

### 3. 色分け(FR-06: 紐づく場合は同系色、フリー/割り込みは別色)

- 紐づく実績(`linked: true`): 現状の sky 系のまま(`bg-sky-600/90` 等)
- 紐づかない実績(`linked: false`。フリータイマー、または紐づき先の予定が見つからない実績): amber 系に変更(`bg-amber-600/90 border-amber-700`、dark:も同様のトーンで用意)。「濃い塗り」の原則(実績は予定より濃い)は維持する
- 色だけに依存しない: 紐づかない実績には日ビュー(`showTime`時)でタイトル下に小さく「フリー」ラベルを表示し、`aria-label` にも常に「(フリー)」を含める(週ビューでラベル非表示でもスクリーンリーダーで判別可能にする)

### 4. ズレの視覚化(FR-06: 開始遅延・超過が視覚的にわかる)

- **開始遅延**(`startDelayMinutes > 0`): 実績レーン内、予定の開始時刻位置(topPercent)から実績ブロックの開始位置までの区間に、装飾用のストライプ柄バー(`aria-hidden`、非クリック)を表示する。日ビューでは「◯分遅れ」ラベルを添える(週ビューは省略、色/柄のみ)
- **超過**(`overrunMinutes > 0`): 実績ブロック自身の末尾(予定の終了時刻以降に対応する部分)を、同じストライプ柄でサブ領域として塗り分ける。日ビューでは「◯分超過」ラベルを添える
- ズレがない場合(delay=0 かつ overrun=0)は装飾なし(通常表示)
- ストライプはCSSの `repeating-linear-gradient` を用いた装飾(色のみに依存しないよう「柄」を追加する ui-quality Skill の方針に従う)
- ズレ情報は装飾の有無に関わらず、実績ブロックの `aria-label` に文言として含める(例: 「設計レビューの実績を編集(開始15分遅れ)」)。これにより色覚多様性・スクリーンリーダー両方に対応する

### 5. 文言(`lib/timer/messages.ts` に追加)

- `delayLabel: (min: number) => \`開始${min}分遅れ\``
- `overrunLabel: (min: number) => \`${min}分超過\``
- `freeBadge: "フリー"`
- 既存の `editLabel` / `startLabel` 系に上記を連結する形でaria-labelを構成する

## スコープ外

- 早期開始・早期終了の可視化(上記参照)
- ギャップサマリー(今日/今週の集計、%表示)は P3-2 で別途仕様化する
- 予定に対応する実績が存在しない(未着手)場合の特別な表示(空白のままとする。P3-2のスコープ)
- タイマー開始/停止・手動編集など既存のインタラクションロジックの変更(本仕様は表示のみが対象)

## テストシナリオ

### 単体(`lib/timer/gap.ts`)

- S1 [単体]: Given 実績が予定と同時刻に開始・終了する, When `computeActualGap` を実行, Then `linked: true, startDelayMinutes: 0, overrunMinutes: 0`
- S2 [単体]: Given 実績開始が対応予定の開始より15分遅い, When 計算, Then `startDelayMinutes: 15`
- S3 [単体]: Given 実績終了が対応予定の終了より20分遅い, When 計算, Then `overrunMinutes: 20`
- S4 [単体・境界値]: Given 実績開始/終了が対応予定より早い(早期開始・早期終了), When 計算, Then `startDelayMinutes: 0, overrunMinutes: 0`(マイナスにならない)
- S5 [単体・異常系]: Given `googleEventId` が `null`(フリータイマー), When 計算, Then `linked: false, startDelayMinutes: 0, overrunMinutes: 0`
- S6 [単体・異常系]: Given `googleEventId` は設定されているが同日の予定に一致するものがない(予定削除等), When 計算, Then `linked: false`
- S7 [単体・境界値]: Given 同じ予定に紐づく実績が2件(中断・再開、時系列順), When 計算, Then 最初の実績のみ `startDelayMinutes` が計算され、最後の実績のみ `overrunMinutes` が計算される(中間はどちらも0)

### 結合(`components/calendar-view.tsx` のレンダリング)

- S8 [結合]: Given 紐づく実績が予定より遅く開始した, When カレンダー表示, Then 実績ブロックに「開始N分遅れ」の文言とストライプ装飾要素が表示される
- S9 [結合]: Given 紐づく実績が予定より遅く終了した, When 表示, Then 「N分超過」の文言とストライプ装飾要素が表示される
- S10 [結合]: Given フリータイマーの実績(`google_event_id` が `null`), When 表示, Then amber系の見た目になり、「フリー」ラベルが表示され、`aria-label` にも「フリー」が含まれる
- S11 [結合・正常系]: Given 予定と実績がズレなく一致, When 表示, Then ズレ装飾(ストライプ・ラベル)が表示されない
- S12 [結合・境界値]: Given 週ビュー(レーンが狭い), When ズレのある実績を表示, Then テキストラベルは表示されないが `aria-label` にはズレ情報が含まれる

### 結合(DB連携)

- S13 [結合]: Given `time_entries` に `google_event_id` が設定された行が存在する, When `fetchTimeEntries` を実行, Then 返り値の各要素に `googleEventId` が含まれる
