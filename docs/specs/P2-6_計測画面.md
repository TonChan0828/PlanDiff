## 目的

カレンダー画面を経由せずに「作業の計測だけを実施・確認できる」画面を追加する。

ユーザー確認済みの動機(2026-07-11):

1. **素早く計測を開始したい** — カレンダーを開かず、画面を開いてすぐタイマーを開始/停止したい
2. **予定なしでも使いたい** — 予定を立てていない日でも記録だけは続けられる入口が欲しい

### コンセプトとの整合(重要)

「記録が目的化」は要件定義書 §1.2 で挙げた Toggl への批判点であり、計測専用画面は放置するとコンセプト(計画 vs 実績のギャップ)から逸脱する。整合は**「予定への昇格導線」**で保つ(ユーザー決定 2026-07-11):

- 予定に紐づかない実績(フリータイマー由来)から、ワンタップで「次回の予定」を作成できる
- これにより「記録する → 記録が次の計画になる → ギャップが見える」というコアループ(§1.3)への還流を画面内に持つ

## 仕様

### 画面・ルーティング

- ルート: `/track`(`(app)` グループ = 認証必須)。ナビ表示名は「計測」
- カレンダー画面ヘッダーのリンク群に「計測」を追加。計測画面からはカレンダー/サマリー/設定への同形式のリンクを置く
- オンボーディング未読時のリダイレクトは既存どおり `/calendar` のみで行う(本画面では行わない。P4-1 の挙動を変えない)

### 画面構成(モバイルファースト、上から順)

1. **タイマー操作エリア**
   - 実行中タイマーあり: タイトル・経過時間(1秒更新。`lib/timer/elapsed` 再利用)・停止ボタンを表示
   - 実行中なし: フリータイマー開始フォーム(タイトル入力+開始ボタン。タイトル空欄可・trim は FR-05 と同一仕様。`FreeTimerBar` と同等)
2. **「今の予定から開始」セクション**
   - 当日の予定のうち「現在進行中」+「これから始まる直近」を最大3件表示し、タップで予定連動タイマーを開始(FR-04 と同一挙動: 既存実行中の自動停止を含む)
   - 該当予定ゼロ件のときはセクションごと非表示(空状態文言は出さない)
3. **「実績」リスト(確認エリア・日付グループ化)**
   - サーバー取得範囲(週±1週間 = 直近2週間相当)の確定済み実績を、ユーザーのタイムゾーンの暦日ごとにグループ化して表示(2026-07-20 拡張。従来は当日のみ)
   - グループは日付の新しい順(上ほど新しい)。当日グループは「今日」ラベルで最上部、過去日は日付ラベル(例: 「7/19(土)」)
   - 各グループ内の実績は開始降順(新しい順)
   - 未来日(取得範囲に混じり得る)の実績は表示しない
   - 各行: タイトル(空タイトルは既存の「(無題)」表記に合わせる)・開始〜終了時刻・所要時間・予定連動/フリーの区別(日付はグループ見出しに出すため行内では時刻のみ)
   - ゼロ件時は空状態メッセージ(例: 「実績はまだありません」)
   - サーバー取得範囲・DBクエリは変更しない(既存 `fetchTimeEntries` の週±1週間をそのまま活用)。「今日」判定・日付グループ化はユーザーTZが必要なためクライアント側の純関数で行う
   - 実績の編集・削除はこの画面では行わず、カレンダー画面(P2-4)への導線リンクを置く
4. **予定への昇格導線**
   - フリー実績(`googleEventId === null`)の行に「予定にする」ボタンを表示
   - タップで予定作成フォーム(P2-5 の作成パネルと同等)を開き、初期値を引き継ぐ:
     - タイトル = 実績タイトル(空なら空のまま。保存時は P2-5 のバリデーションどおり必須)
     - 開始 = **翌日の同時刻**、終了 = 開始 + 実績の所要時間
   - ユーザーが編集して保存すると `createAppEventAction`(P2-5)でアプリ内予定を作成する
   - 予定連動実績(既に予定がある)には表示しない

### データ取得・更新

- 新規テーブル・カラム・マイグレーションは**なし**(既存の `time_entries` / `synced_events` のみ)
- 読み取り(Server Component): `fetchRunningEntry` / `fetchTimeEntries`(基準日=現在)/ `fetchSyncedEvents`(基準日=現在)を再利用。「今日」のフィルタと「進行中+直近」の抽出はユーザーTZが必要なためクライアント側の純関数で行う(既存カレンダーのTZ方針を踏襲)
- 書き込み(Server Action): `startTimerAction` / `stopTimerAction` / `createAppEventAction` を再利用。**各アクションの `revalidatePath` に `/track` を追加する**(現状 `/calendar` のみのため、計測画面の表示が更新されない)
- 抽出ロジック(純関数・新規):
  - `selectQuickStartEvents(events, now)`: 進行中(`startAt <= now < endAt`)を先頭に、以降は `now` 以降に開始する当日の予定を開始昇順で、合計最大3件。`startAt === now` は進行中扱い、`endAt === now` は除外
  - `filterTodayEntries(entries, now, timeZone)`: ユーザーTZの「今日」に開始する確定済み実績を開始降順で返す
  - `buildPromotionDefaults(entry, timeZone)`: 昇格フォームの初期値(翌日同時刻+同じ長さ)を返す。日時演算は date-fns を使う

### エラー時の挙動

- タイマー開始/停止・予定作成の失敗時は既存パターンどおり日本語のエラーメッセージを画面内に表示する(例: 「タイマーを開始できませんでした」)。メッセージ定数は `lib/track/messages.ts` に集約
- 実績・予定の読み込み失敗は既存ヘルパーの例外メッセージに従う

## スコープ外

- ミニギャップサマリーの表示(ギャップ確認はサマリー画面に委ねる)
- 実績の編集・削除(P2-4 のカレンダー画面に委ねる。導線リンクのみ)
- 取得範囲(週±1週間)を超える過去実績の閲覧・ページング(それより前はカレンダー/サマリーで)
- 予定連動実績の昇格・繰り返し予定化
- 計測画面のホーム(ログイン後初期画面)化
- Google 連携関連の UI(凍結中。P2-5 のフラグ方針を変えない)

## テストシナリオ

### 単体

- S1 [単体]: Given 現在進行中の予定1件・未来の予定3件・過去の予定1件 When `selectQuickStartEvents` Then 進行中が先頭、以降は開始が近い順で合計3件(過去は含まない)
- S2 [単体]: Given `startAt === now` の予定と `endAt === now` の予定 When `selectQuickStartEvents` Then 前者は進行中として含まれ、後者は含まれない(境界値)
- S3 [単体]: Given 昨日・今日・明日に開始した確定済み実績 When `groupEntriesByDay`(ローカルTZ) Then 未来(明日)は除外され、今日・昨日が新しい日付順のグループになり、当日グループの `isToday` が true
- S4 [単体]: Given 開始 10:00〜終了 11:30 のフリー実績 When `buildPromotionDefaults` Then 翌日 10:00〜11:30 の初期値が返る
- S5 [単体]: Given 実行中タイマーなし When 計測画面を描画 Then フリータイマー開始フォームが表示され、開始押下で `onStart` が呼ばれる
- S6 [単体]: Given 実行中タイマーあり When 計測画面を描画 Then タイトル・経過時間・停止ボタンが表示され、停止押下で `onStop` が呼ばれる
- S7 [単体]: Given フリー実績と予定連動実績が並ぶ実績リスト When 描画 Then 「予定にする」ボタンはフリー実績の行にのみ表示される
- S8 [単体]: Given 今日の実績ゼロ件 When 描画 Then 空状態メッセージが表示される
- S9 [単体]: Given タイマー開始アクションが `{ ok: false }` を返す When 開始を押す Then 日本語のエラーメッセージが表示される

### 結合

- S10 [結合]: Given 実行中タイマー1件と当日実績2件がDBにある状態 When `/track` を Server Component として描画(〜DB) Then 実行中カードと当日実績一覧が表示される
- S11 [結合]: Given 実行中タイマーAがある状態 When 計測画面から予定連動タイマーBを開始(`startTimerAction`〜DB) Then Aは自動停止して実績化され、Bが実行中になる(実行中は常に1本)
- S12 [結合]: Given フリー実績1件 When 昇格フォームを送信(`createAppEventAction`〜DB) Then `synced_events` に `source='app'`・引き継いだタイトル/翌日日時の行が作成される

### 過去実績の日付グループ表示(2026-07-20 拡張)

- S13 [単体]: Given 複数日(今日・昨日・一昨日)にまたがる確定済み実績 When `groupEntriesByDay` Then 日付の新しい順にグループ化され、各グループ内は開始降順で並ぶ
- S14 [単体]: Given 今日と過去日の実績 When `groupEntriesByDay` Then 当日グループのみ `isToday` が true、過去日グループは false
- S15 [単体・境界]: Given 翌日(未来)に開始した実績 When `groupEntriesByDay` Then その実績はどのグループにも含まれない
- S16 [単体]: Given 実績ゼロ件 When `groupEntriesByDay` Then 空配列を返す
- S17 [単体・コンポーネント]: Given 今日と過去日の実績を渡す When 計測画面を描画 Then 「今日」見出しと過去日の日付見出しに分かれ、過去日の実績も表示される
