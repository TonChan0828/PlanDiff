# 仕様書: P5-3 サマリー件数ステータス行

- ステータス: 承認済み(2026-07-18)
- 関連: docs/要件定義書.md の FR-07(ギャップサマリー)/ docs/specs/P3-2_ギャップサマリー.md / docs/specs/D-3_サマリーヒーローとLP.md
- 指示資料: `docs/指示資料/` に本タスク関連の資料なし(README のみ。2026-07-18 確認)

## 目的

サマリーページ(`/summary`)で「今の期間(今日/今週)の振り返り」を深めるため、
算出済みの件数情報(予定件数・着手・未着手・割り込み件数と合計時間)を
diffヒーロー直下に1行のステータス行として表示する。
ページを開いた瞬間に「予定のうちどれだけ手を付けたか・割り込みがどれだけあったか」を把握できるようにする。

ユーザーとの検討経緯(2026-07-18):
- 方向性 = 「今の期間の振り返りを深める」(v2の係数分析・期間拡張はやらない)
- 追加項目 = 件数系の集計のみ(開始遅延・予定単位%は今回見送り)
- 表示位置 = ヒーロー直下のステータス行(セクション見出しへの件数追加はしない)

## 仕様

### 表示イメージ

```
今日のズレ
  +0:55
  ▔▔▔▔▔▔
計画 6:00 / 実績 6:55 (+15%)
予定5件・着手3・未着手2・割り込み2件(1時間20分)   ← 新規追加(ステータス行)
```

### データ(集計)

`lib/summary/aggregate.ts` の `GapSummary` に以下のフィールドを追加する
(いずれも既存の `items` / `interruptions` から導出。集計対象・範囲判定は P3-2 から変更しない):

| フィールド | 定義 |
|---|---|
| `planCount` | `items.length`(範囲内の予定件数) |
| `startedCount` | `items` のうち `actualMinutes > 0` の件数 |
| `notStartedCount` | `items` のうち `notStarted === true` の件数(`startedCount + notStartedCount === planCount`) |
| `interruptionCount` | `interruptions.length` |
| `interruptionTotalMinutes` | `interruptions` の `actualMinutes` 合計 |

### 表示(フォーマット)

- `lib/summary/format.ts` に `formatSummaryCounts(summary の件数フィールド): string` を追加する
  - 通常: `予定{n}件・着手{m}・未着手{k}・割り込み{j}件({合計時間})`
    - 合計時間は既存 `formatDurationMinutes` で「1時間20分」形式
  - 予定0件のとき: 着手・未着手を省略し `予定0件・…`
  - 割り込み0件のとき: `…・割り込みなし`(時間表記は省略)
  - 例: 予定0件・割り込み0件 → `予定0件・割り込みなし`
- 文言(「予定」「件」「着手」「未着手」「割り込み」「割り込みなし」「・」等)は
  `lib/summary/messages.ts` の `SUMMARY_MESSAGES` に追加し、ハードコードしない
- `app/(app)/summary/page.tsx` の diffヒーロー内、メタ行(`data-testid="gap-hero-meta"`)の直下に
  `data-testid="gap-hero-counts"` の1行として描画する
  - スタイルはメタ行に準拠(`text-ink-muted` の小さめテキスト。数字は `tabular-nums`)
  - 375px 幅で収まらない場合は自然に折り返す(横スクロールを発生させない)

### データ変更・エラー時の挙動

- DBスキーマ変更なし。新規データ取得なし(既存 `computeGapSummary` の入力は不変)
- エラーハンドリングの追加なし(既存のページのエラー挙動を変えない)

## スコープ外

- 開始遅延(予定開始と着手時刻の差)の表示 → 将来検討
- 予定単位のズレ%表示 → 将来検討
- セクション見出し(「予定ごとの内訳」「割り込み・フリー作業」)への件数追加
- 期間の拡張(月次等)、按分、ソート・フィルタ(P3-2 のスコープ外事項を維持)
- 見積もり係数などの傾向分析(v2)

## テストシナリオ

対応テストファイル: `tests/lib/summary/aggregate.test.ts` / `tests/lib/summary/format.test.ts` /
`tests/app/(app)/summary/page.test.tsx` / `tests/integration/summary.test.ts`(既存構造にならう)

### 正常系

- S1 [単体]: Given 範囲内に予定2件(片方のみ実績あり)と割り込み実績2件(30分+50分) When `computeGapSummary` を実行 Then `planCount=2` / `startedCount=1` / `notStartedCount=1` / `interruptionCount=2` / `interruptionTotalMinutes=80` を返す
- S2 [単体]: Given 予定5件・着手3・未着手2・割り込み2件(80分) When `formatSummaryCounts` を実行 Then `予定5件・着手3・未着手2・割り込み2件(1時間20分)` を返す
- S3 [単体]: Given 予定・実績・割り込みが存在する状態でサマリーページを描画 When ヒーローを表示 Then `gap-hero-counts` がメタ行の直下に表示され、件数の文言が集計結果と一致する

### 境界値

- S4 [単体]: Given 予定0件・割り込み1件(30分) When `formatSummaryCounts` を実行 Then `予定0件・割り込み1件(30分)` を返す(着手・未着手を省略)
- S5 [単体]: Given 予定3件(着手1・未着手2)・割り込み0件 When `formatSummaryCounts` を実行 Then `予定3件・着手1・未着手2・割り込みなし` を返す(時間表記を省略)
- S6 [単体]: Given 予定0件・割り込み0件 When `formatSummaryCounts` を実行 Then `予定0件・割り込みなし` を返す

### 異常系(変則データ)

- S7 [単体]: Given 開始=終了の0分実績(予定紐づき1件・割り込み1件) When `computeGapSummary` を実行 Then 予定紐づき0分実績の予定は `startedCount` に含めず(`notStarted` のまま)、0分割り込みは `interruptionCount=1` / `interruptionTotalMinutes=0` として矛盾なく集計される

### 結合

- S8 [結合]: Given DBに範囲内の予定2件・うち1件に紐づく実績1件・予定に紐づかない実績1件(30分)を投入 When サマリーページをサーバー描画 Then ステータス行に `予定2件・着手1・未着手1・割り込み1件(30分)` が表示される
