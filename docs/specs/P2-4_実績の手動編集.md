## 目的

カレンダー画面の実績レーン(右45%)に表示されている**確定済み**実績(`end_at IS NULL` でない = 実行中でないもの)を、
タップして開始/終了時刻とタイトルを修正、または削除できるようにする。
記録漏れ・記録ミス(打刻し忘れ、タイトルの入れ間違い)を後から直せることがギャップ分析(P3-2)の前提になる。

## 仕様

- 新規マイグレーションは**なし**。`time_entries` の update/delete RLSポリシー(`time_entries_update_own` / `time_entries_delete_own`)は P0-6 で作成済みで、そのまま使う
- Google APIは使わない

### 対象と編集不可のケース

- 編集・削除の対象は**確定済み実績のみ**(`end_at` が設定されている行)
- **実行中エントリ(`end_at IS NULL`)は編集不可**。実績レーンの実行中ブロックはタップしても何も起きない(停止してから編集する)
- サーバー側(`updateTimeEntry` / `deleteTimeEntry`)でも `end_at IS NOT NULL` を条件に含め、実行中エントリを直接操作できないよう二重に防ぐ
- 対象は現在表示中(取得済み)の実績のみ。表示範囲外(表示週±1週間の外)の実績はこの画面から編集できない(範囲を変えて表示すれば編集可能)

### 操作(実績ブロックタップ)

- 実績レーンの確定済みブロックは `<button>` 化し、タップで編集パネル(モーダル)を開く
- 編集パネルは画面中央に固定表示するモーダル(`role="dialog"` `aria-modal="true"`)。背景をディム(`bg-black/40`)し、背景タップまたは閉じるボタンでキャンセルできる
- パネルの内容: タイトル入力、開始時刻・終了時刻(`DateTimeStepper`。P5-5)、削除ボタン、保存ボタン、閉じるボタン

### タイトル

- フリータイマー(P2-3)と同じ規約: 空欄可、送信直前に `trim()`。文字数上限なし

### 開始/終了時刻

- `DateTimeStepper`(日付+時▲▼:分▲▼。P5-5)で編集する(端末ローカルタイムゾーンでの入出力。表示・保存はUTCへ変換。分00→59/59→00の桁跨ぎで時も連動する)
- バリデーション(クライアント側、送信前):
  - 開始・終了のどちらかが未入力 → 送信せず「開始・終了時刻を入力してください」を表示
  - 終了時刻が開始時刻より前 → 送信せず「終了時刻は開始時刻以降にしてください」を表示(等しい場合は許可。P2-2の0分エントリと同じ扱い)
- サーバー側は `time_entries_time_range` のCHECK制約(`end_at >= start_at`)が最終防衛線。制約違反時はDBエラーとなり `updateTimeEntry` が失敗を返す

### Server Action / データ変更

**updateTimeEntry(id, { title, startAt, endAt })**

1. 認証確認
2. `time_entries` を `id` かつ `end_at IS NOT NULL` の条件で `title` / `start_at` / `end_at` を更新(RLSにより他人の行は対象にならない)
3. 対象行が0件(存在しない/他人の行/実行中/未認証)の場合は失敗を返す

**deleteTimeEntry(id)**

1. 認証確認
2. `time_entries` を `id` かつ `end_at IS NOT NULL` の条件で削除
3. 対象行が0件の場合は失敗を返す

- 時刻はクライアントの `DateTimeStepper`(P5-5)入力をローカルタイムゾーンとして解釈し、UTCのISO文字列に変換してからServer Actionへ渡す(表示側の変換のみ。桁跨ぎ計算は `lib/ui/local-date-time` の date-fns ベース純関数で行い、直接のDate演算はしない)
- 削除に確認ステップを設ける(下記「削除の確認」)。誤操作防止のため、削除は即実行しない

### 削除の確認

- 編集パネルの「削除」ボタンをタップすると、パネル内の表示が確認表示(「この実績を削除しますか?」+「削除する」+「キャンセル」)に切り替わる(ブラウザネイティブの `confirm()` は使わない)
- 「削除する」で `deleteTimeEntryAction` を呼ぶ。「キャンセル」で編集フォーム表示に戻る

### 保存/削除の成否と反映

| 操作 | 成功時 | 失敗時 |
|---|---|---|
| 保存 | パネルを閉じ、`router.refresh()` で最新表示に反映 | パネルは開いたまま、入力値を保持し、日本語エラー「実績を更新できませんでした」を表示 |
| 削除 | パネルを閉じ、`router.refresh()` で最新表示に反映 | 確認表示のまま、日本語エラー「実績を削除できませんでした」を表示 |
| 未認証(セッション切れ) | — | `/login` へリダイレクト(既存Server Actionと同じ) |

- 保存・削除の処理中はボタンを無効化する(連打対策。既存のタイマー操作と同じ方針)
- 実行中タイマー(P2-2/P2-3のUI)には影響しない。楽観的更新は行わない(編集は即時性が求められる操作ではないため。CLAUDE.mdのアーキテクチャ原則3に基づき、確定はServer Action成功後の`router.refresh()`に委ねる)

### アクセシビリティ / UI品質(ui-quality Skill準拠)

- 実績ブロックの `button` 化: `aria-label="{タイトルまたは(タイトルなし)}の実績を編集"`
- パネルは `role="dialog"` `aria-modal="true"` `aria-label="実績を編集"`
- 各入力に `label`(タイトル/開始時刻/終了時刻)を関連付ける
- ボタン(保存/削除/削除する/キャンセル/閉じる)はすべて最小タップターゲット44px
- 削除系ボタンは色だけに頼らず「削除」というラベルテキストを併記する
- 375px幅でパネルが画面内に収まり横スクロールが出ないこと(`max-w` + 左右余白)
- dark: 対応、コントラスト比4.5:1以上(既存規約どおり)

## スコープ外

- 実行中エントリの編集(停止してから編集する。P2-2/P2-3の停止操作を使う)
- 表示範囲外(週±1週間の外)の実績の一覧・編集(範囲を移動して表示すれば対象になる)
- 削除の取り消し(Undo)
- 複数件の一括編集・一括削除
- 予定との紐付け(`google_event_id`)の変更・新規紐付け
- オーバーレイ表示の作り込み(P3-1)

### 単体(components/edit-entry-panel)

- S2 [単体] 正常系: Given 実績エントリ(タイトル・開始・終了) When パネルを描画 Then タイトル入力と開始/終了の`DateTimeStepper`(P5-5)に初期値が反映される
- S3 [単体] 正常系: Given タイトル前後に空白を含む入力 When 保存 Then `onSave` がトリム済みのタイトルとISO(UTC)の`startAt`/`endAt`で呼ばれる
- S4 [単体] 異常系(境界値): Given 終了時刻が開始時刻より前 When 保存 Then `onSave` は呼ばれず「終了時刻は開始時刻以降にしてください」が表示される
- S5 [単体] 境界値: Given 終了時刻が開始時刻と同じ When 保存 Then `onSave` が呼ばれる(等しい場合は許可)
- S6 [単体] 異常系: Given 開始または終了が未入力 When 保存 Then `onSave` は呼ばれず「開始・終了時刻を入力してください」が表示される
- S7 [単体] 正常系: Given 「削除」ボタンをタップ When 描画更新 Then 確認表示(「この実績を削除しますか?」+「削除する」+「キャンセル」)に切り替わり、「キャンセル」で編集フォームに戻る。「削除する」で `onDelete` が呼ばれる
- S8 [単体] 正常系: Given `pending=true` When 描画 Then 保存・削除関連のボタンがすべて無効化される
- S9 [単体] 正常系: Given `error` prop が設定されている When 描画 Then エラーメッセージが表示される

### 単体(calendar-view統合)

- S10 [単体] 正常系: Given 確定済み実績ブロック(`editable: true`) When タップ Then 編集パネルが開き、対象エントリの内容が渡される
- S11 [単体] 正常系: Given 実行中の実績ブロック(`editable: false`) When タップ Then 何も起きない(編集パネルは開かない)
- S12 [単体] 正常系: Given 編集パネルで保存 When `updateTimeEntryAction` が成功を返す Then パネルが閉じ、`router.refresh()` が呼ばれる
- S13 [単体] 異常系: Given 編集パネルで保存 When `updateTimeEntryAction` が失敗を返す Then パネルは開いたままエラー「実績を更新できませんでした」が表示される
- S14 [単体] 正常系: Given 編集パネルで削除確定 When `deleteTimeEntryAction` が成功を返す Then パネルが閉じ、`router.refresh()` が呼ばれる
- S15 [単体] 異常系: Given 編集パネルで削除確定 When `deleteTimeEntryAction` が失敗を返す Then 確認表示のままエラー「実績を削除できませんでした」が表示される

### 結合(lib/timer/service 〜 ローカルDB。RLS認証済みクライアントで実行)

- S16 [結合] 正常系: Given 確定済み実績 When `updateTimeEntry` でタイトル・開始・終了を変更 Then DB上の行が更新される
- S17 [結合] 異常系(境界値): Given 確定済み実績 When `updateTimeEntry` で `endAt < startAt` を指定 Then 失敗を返し、DBの行は変更されない(CHECK制約)
- S18 [結合] 異常系: Given 実行中エントリ(`end_at IS NULL`) When `updateTimeEntry` を実行 Then 失敗を返し、実行中エントリは変更されない
- S19 [結合] 正常系: Given 確定済み実績 When `deleteTimeEntry` Then DBから行が削除される
- S20 [結合] 異常系: Given 実行中エントリ When `deleteTimeEntry` を実行 Then 失敗を返し、実行中エントリは削除されない
- S21 [結合] 境界値(分離): Given ユーザーBの確定済み実績 When ユーザーAが同じidで `updateTimeEntry` / `deleteTimeEntry` を実行 Then 失敗を返し、Bの行は変更・削除されない(RLS)
