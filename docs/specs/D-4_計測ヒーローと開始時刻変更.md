  デザインモック: https://claude.ai/code/artifact/64ba1855-fcee-441f-93f1-f79dfc468eef

## 目的

計測中の経過時間が小さく(14px)、計測画面でも脇役のままで「いま計測している」
実感が薄い。経過時間を計測画面の主役(ヒーロー)に据えて視認性を上げる。
また、タイマーの開始を忘れて後から気づくケースで、いったん停止して実績を
編集し直す手間があるため、**計測中に開始時刻をその場で変更**できるようにする。

## 仕様

### 1. 計測ヒーロー(/track・実行中のみ)

計測画面の最上部(見出し直下)に、実行中タイマーのヒーローカードを表示する。
実行中でない場合は従来どおりフリータイマーバーを表示する(変更なし)。

```
┌────────────────────────────────┐
│ ● 記録中                        │
│                                │
│         1:09:24                │ ← 経過時間。等幅・特大(約56px)。
│      本番障害の対応               │    秒は淡色にして桁の暴れを抑える
│                                │
│ [開始 11:32 を変更]   [停止]     │ ← 開始時刻ボタン+大型停止ボタン
└────────────────────────────────┘
```

- 経過時間: Geist Mono・tabular・`text-5xl`超(375pxで約56px)。中央寄せ。
  秒部分(`:SS`)は `text-ink-muted` で一段引く。1秒ごと更新(既存ロジック)
- 記録中ドット(柿・点滅。`prefers-reduced-motion` で停止)と「記録中」ラベル
- タイトル(なければ「(タイトルなし)」)を経過時間の直下に表示
- **開始時刻ボタン**: 「開始 11:32」(等幅)をタップすると開始時刻変更パネルを
  開く。ボタンとして44px以上を確保
- **停止ボタン**: 紅・大型(min-h-12)。挙動は既存の停止と同じ
- カレンダー画面の実行中バー(RunningTimerBar)は据え置きつつ、
  経過時間を `text-sm` → `text-lg` に拡大し、開始時刻(タップで同じ変更
  パネルを開く)を追加する

### 2. 開始時刻変更パネル(実行中のみ)

- モーダル(実績編集パネルP2-4と同じ作法)。フィールドは**開始時刻のみ**
  (`datetime-local`)。初期値は実行中エントリの開始時刻(ローカルTZ)
- 保存で Server Action `updateRunningStartAction(startAtIso)` を呼ぶ。
  成功でパネルを閉じ `router.refresh()`(経過時間は新しい開始時刻で再計算される)
- バリデーション(クライアント/サーバー両方):
  - 未入力 → 「開始時刻を入力してください」
  - 未来時刻 → 「開始時刻は現在より後にできません」
    (サーバーは時計ズレ許容として**+60秒まで**を「現在」とみなす)
- 失敗時 → 「開始時刻を変更できませんでした」(パネル内に表示、閉じない)

### 3. サーバー(service / Server Action)

- `lib/timer/service.ts` に `updateRunningStart(client, startAtIso)` を追加:
  - `end_at IS NULL` の行のみを対象に `start_at` を更新(`.is("end_at", null)`)。
    確定済み実績には一切影響しない(既存 `updateTimeEntry` は逆に実行中を除外
    しており、対称になる)
  - 対象行が無ければ `{ ok: false }`(実行中なし=停止直後の競合など)
  - `startAt` が不正なISO・サーバー現在時刻+60秒より未来なら `{ ok: false }`
  - RLSにより他ユーザーの行は対象外(既存ポリシーのまま。スキーマ変更なし)
- `updateRunningStartAction` は既存Actionと同じ作法
  (認証→service→`revalidatePath("/calendar")`+`/track`)

### 4. 文言(lib/timer/messages.ts に追加)

| キー | 文言 |
|---|---|
| runningSinceLabel | `開始 {HH:mm} を変更`(aria-label兼用) |
| editStartTitle | 開始時刻を変更 |
| editStartRequired | 開始時刻を入力してください |
| editStartFuture | 開始時刻は現在より後にできません |
| editStartError | 開始時刻を変更できませんでした |

## スコープ外

- 実行中エントリのタイトル変更・終了時刻の先回り入力(停止時に確定する現仕様を維持)
- 確定済み実績の編集(既存P2-4のまま)
- DBスキーマ・RLSの変更(なし)
- カレンダーの実行中「ブロック」タップ挙動の変更(タップ=停止のまま)

## テストシナリオ

- S1 [単体]: Given 実行中タイマー When /track を描画 Then ヒーローに経過時間・
  タイトル・「開始 HH:mm を変更」ボタン・停止ボタンが表示される
- S2 [単体]: Given ヒーロー表示 When 開始時刻ボタンをタップ Then パネルが開き、
  初期値が現在の開始時刻になっている
- S3 [単体]: Given パネル When 未来時刻を入力して保存 Then 日本語エラーが表示され
  Actionは呼ばれない(異常系)
- S4 [単体]: Given パネル When 空欄で保存 Then 「開始時刻を入力してください」
  (境界値)
- S5 [単体]: Given パネル When 有効な過去時刻で保存 Then Actionが呼ばれ、
  成功でパネルが閉じる。失敗時はエラー表示のままパネルは閉じない
- S6 [単体]: Given カレンダーの実行中バー When 描画 Then 拡大された経過時間と
  開始時刻ボタンが表示され、タップで同じパネルが開く
- S7 [結合]: updateRunningStart で実行中エントリの start_at が更新される
- S8 [結合]: 実行中エントリが無い場合は ok:false。確定済み実績の start_at は
  変更されない(対象外の確認)
- S9 [結合]: サーバー現在時刻+60秒より未来の startAt は拒否される(ok:false)。
  現在時刻ちょうどは許容(境界値)
- S10 [結合]: RLS: 他ユーザーの実行中エントリは更新されない
