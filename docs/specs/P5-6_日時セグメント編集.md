## 目的

P5-5 の `DateTimeStepper` は「ネイティブ `<input type="date">` + 時[▲▼] + 分[▲▼]」の3ブロック構成。
各ブロックの間に▲▼ボタンがTAB順に挟まるため、PCで「日付→時→分」を修正するたびに
TABを何度も押す必要があり手間、というユーザー要望が出た。

ネイティブの日時入力と同じ操作感 —— **← → でセグメント移動、↑↓ で増減、数字で自動送り** ——
に寄せ、**TABを不要にする**。日付も独自セグメント化して 年/月/日/時/分 をひとつの入力群にまとめ、
← → で全セグメントをシームレスに移動できるようにする。モバイル向けにカレンダーの
ポップアップ選択(ネイティブ日付ピッカー)はボタンとして残す。

props 契約(`label` / `value`(`"yyyy-MM-dd'T'HH:mm"` または `""`)/ `onChange` / `disabled`)は
**変更しない**。よって適用先3パネル(実績編集・開始時刻変更・アプリ内予定/定期予定編集)は無改修。
DBスキーマ・RLS変更なし。

## 仕様

### 1. 純関数: `lib/ui/local-date-time.ts`(拡張)

既存の `parseLocalDateTime` / `formatLocalDateTime` / `stepLocalHour` / `stepLocalMinute` /
`setLocalHour` / `setLocalMinute` / `setLocalDate` はそのまま残す(既存テスト S1〜S14 維持)。
以下を追加する。**Date 直接演算はせず date-fns 経由**(月末・年跨ぎ・うるう年を委譲)。

```ts
export type LocalSegment = "year" | "month" | "day" | "hour" | "minute";

// セグメントを ±1。桁跨ぎ連動:
//   minute 59→+1 で hour+1 / 00→-1 で hour-1、hour で日跨ぎ、day で月跨ぎ、month で年跨ぎ。
//   year は上位桁がないため year のみ変化。addYears/addMonths/addDays/addHours/addMinutes を使う。
export function stepLocalSegment(value: string, segment: LocalSegment, delta: 1 | -1): string;

// セグメントへ数値を設定(直接入力の確定用)。桁は跨がずクランプする:
//   month 1–12 / hour 0–23 / minute 0–59 / day はその年月の日数でクランプ(例 2月31→28/29) /
//   year は 1–9999 でクランプ。無効・空 value はそのまま返す。
export function setLocalSegment(value: string, segment: LocalSegment, n: number): string;
```

- `value` が空文字列または不正形式の場合、両関数とも `value` をそのまま返す(副作用なし)
- DST は対象外(親仕様と同じ。日本=JSTを主対象)

### 2. コンポーネント: `components/date-time-stepper.tsx`(作り替え)

現行の3ブロック構成を、**1つの枠内に 年/月/日 時:分 の5セグメントを並べた統合フィールド**へ置換する。

#### 構造

```
<fieldset disabled={disabled}>
  <legend>{label}</legend>
  <div role="group" aria-label={label}>            // 枠(border, rounded)
    [年4桁] / [月2桁] / [日2桁]  [時2桁] : [分2桁]   // 各セグメント=spinbutton input
    [▲/▼]                                            // フォーカス中セグメントに作用する共有ステッパー
    [📅]                                             // カレンダーボタン
  </div>
  <input type="date" class="sr-only" tabIndex=-1 /> // 視覚的に隠したピッカー用
</fieldset>
```

- 各セグメントは `role="spinbutton"` の `<input inputMode="numeric">`。
  `aria-label`(`${label}の年/月/日/時/分`)、`aria-valuemin/max/now` を持つ。
  常にゼロ埋め(年4桁・他2桁)、`font-mono tabular-nums`。区切り(`/`・空白・`:`)は `aria-hidden`
- セパレータ・幅は 375px で枠内に収まるよう固定幅(年 `w-12` 相当、他 `w-8` 相当)+ `flex-nowrap`

#### キーボード操作(各セグメント `onKeyDown`。該当キーは `preventDefault`)

- **ArrowLeft / ArrowRight**: 前/後のセグメントへ**フォーカス移動**。端(年で←、分で→)では移動しない
- **ArrowUp / ArrowDown**: フォーカス中セグメントを `stepLocalSegment(value, segment, ±1)` で増減
  (**桁跨ぎあり**。分59→時+1、日で月跨ぎ 等)。移動後もフォーカスは維持
- **数字キー(0–9)**: セグメントの「入力バッファ」に桁を積み、`setLocalSegment` でクランプ確定する。
  次の条件で**次セグメントへ自動送り**(ネイティブ準拠):
  - バッファが桁数上限(年=4・他=2)に達したとき、または
  - 1桁目 `d` について「もう1桁入れても範囲に収まらない」とき(例 分/日で `d≥6`・`d≥4`、時で `d≥3`、月で `d≥2` → その桁のみで確定して送る)
  - セグメントへフォーカスが入った直後・移動直後はバッファをリセットする
- **Backspace**: バッファ/表示桁をクリア(値は据え置き。実装簡潔化のため必須挙動は上記4種)
- **Enter**: フォーカスを外す(`blur`)。フォームは送信しない

#### 共有▲▼ / カレンダー / disabled

- 共有 **▲▼**: 直近にフォーカスした(または操作対象の)`activeSegment` を `stepLocalSegment` で ±1。
  初期 `activeSegment` は `"minute"`。タップ領域 `min-h-11 min-w-11`(タッチのアフォーダンス維持)
- **カレンダーボタン**(`aria-label="${label}をカレンダーで選ぶ"`): クリックで隠した
  `<input type="date">`(`datePart` にバインド)の `showPicker()` を呼ぶ。`showPicker` 非対応時は
  当該 input へ `focus()` フォールバック。ピッカー側 `onChange` は `setLocalDate` で
  **日付だけ差し替え**(時分は維持)
- **`value===""`(未入力)**: 全セグメントはプレースホルダ(`----/--/-- --:--`)表示で `disabled`、
  共有▲▼も `disabled`。**カレンダーボタンのみ有効**とし、ピッカーで日付を選ぶと
  `onChange("${選択日}T00:00")` で確定する(空からの復帰導線)
- `disabled`(親の pending 等)時は `<fieldset disabled>` で全体を無効化

#### 文言: `lib/ui/messages.ts`(拡張)

`DATE_TIME_STEPPER_MESSAGES` に以下を追加/一般化する(i18nを見据え1箇所に集約):
- `yearLabel/monthLabel/dayLabel`(既存 `hourLabel/minuteLabel` と揃える)
- `incrementLabel/decrementLabel` を年/月/日/時/分に一般化(`unit` の型を拡張)
- `calendarButtonLabel: (label) => \`${label}をカレンダーで選ぶ\``

## スコープ外

- `components/recurring-rule-panel.tsx` の `type="time"`(時刻のみ入力)の置き換え(親仕様と同じく対象外)
- DST を考慮した桁跨ぎ(日本時間を主対象とするため非対応)
- DBスキーマ・RLSの変更(なし)
- 秒単位の入力(分単位のまま)
- **セグメント操作による「空(`""`)への明示クリア**」: 3パネルは常に非空の値で開くため不要。
  空からの復帰はカレンダーボタンで担保する(P5-5 の S22「日付クリアで空」は本仕様で廃止)
