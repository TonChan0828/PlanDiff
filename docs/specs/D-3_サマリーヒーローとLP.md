## 目的

- サマリー画面の主役である「ズレ」が3枚タイルの1枚に埋もれているため、
  計測ヒーロー(D-4)と対になる**diffヒーロー**として昇格させ、
  「今日どれだけズレたか」が一目でわかるようにする
- 未ログインのトップページが仮置きのままのため、新アイデンティティ
  「時間のdiff」でLP(P4-3)を作り、ベータ投入(P4-6)への導線を成立させる

## 仕様

### D-3a. サマリーのdiffヒーロー

期間タブ(今日/今週)の直下、現在の3枚タイルを置き換える。

```
┌──────────────────────────────┐
│ 今日のズレ                     │
│   +0:55                      │ ← 符号付き等幅・特大(約48px)
│   ▨▨▨▨▨▨(ハッチの下線)        │ ← ズレの紋章。ズレ0なら表示しない
│   計画 4:05 / 実績 3:10(+22%) │ ← 等幅・コンパクト
└──────────────────────────────┘
```

- **ズレ値**: `formatSignedDurationMinutes` を「H:MM」形式の符号付きに拡張した
  表示(例 `+0:55` / `-1:10` / `±0:00`)。等幅・tabular・特大(約48px)
- **色**: 正(実績が計画より多い)=柿、負(少ない)=群青、ゼロ=墨。
  色だけに頼らず符号(+/−/±)を必ず伴う(ui-quality)
- **ハッチ下線**: ズレが0でないときのみ、値の下に柿の斜線ハッチ
  (オーバーレイと同じ紋章)を敷く。装飾のため `aria-hidden`
- **見出し**: タブに応じて「今日のズレ」/「今週のズレ」
- 計画合計・実績合計・ズレ%は1行のコンパクト表記に集約(タイル3枚は廃止)
- 「予定ごとの内訳」「割り込み・フリー作業」セクションは現状維持

### D-3b. LP(`app/(marketing)/page.tsx` の置き換え)

新アイデンティティ「時間のdiff」でLPを実装する。**「予定 vs 実績のギャップ」を
主語**にし、自動計測ツール(WakaTime等)と誤解されないコピーにする(要件定義書リスク)。

構成(上から):

1. **ヒーロー**: ワードマーク+見出し「見積もりが、当たるようになる。」+
   サブコピー(予定と実績をひとつのタイムラインに重ね、計画と現実のズレを
   可視化する、の趣旨)+CTA2つ(「無料で始める」→ `/signup`、
   「ログイン」→ `/login`)+**オーバーレイのCSSイラスト**
   (予定=群青10%レーン/実績=群青レーン/ズレ=柿ハッチ+`+17分 遅れ` ラベル。
   画像ファイルではなくトークンで描くCSS製。ダークにも追随)
2. **3つの機能**: (1)計画×実績のオーバーレイ(ズレはハッチと符号付き数値で)、
   (2)ワンタップ計測(タップで開始/停止。自動計測ではないことが伝わる表現)、
   (3)日/週のギャップサマリー(`+0:55` のdiff表記)
3. **使い方3ステップ**(実際の手順という順序情報があるため番号を使う):
   予定を置く → タイマーで記録する → ズレを読む
4. **ベータ案内**: 現在ベータ版・無料で利用可。Googleカレンダー連携は準備中
   (凍結中の機能を「提供中」と書かない)
5. **CTA再掲**+既存フッター(プライバシーポリシー・利用規約は
   marketingレイアウトのまま)

- `metadata` の title / description をLPの内容に更新する
- モバイルファースト(375px)で崩れないこと。ダーク対応(トークン準拠)
- 計測・課金・複数カレンダー等のスコープ外機能を謳わない

## スコープ外

- 料金ページ(P4-4)・興味クリック計測
- LPへの実写スクリーンショット画像の埋め込み(CSSイラストで代替。
  実写が必要になったら別途判断)
- サマリーの内訳・割り込みセクションの構造変更
- 認証まわり・機能の変更(表示のみ)

## テストシナリオ

- S1 [単体]: Given 計画4:05・実績3:10(今日)When サマリー描画
  Then ヒーローに「今日のズレ」と符号付き特大値(-0:55)・
  計画/実績/百分率の行が表示される
- S2 [単体]: Given 実績が計画を上回る When 描画 Then 値が `+` 表記になり
  超過側(柿)のスタイルとハッチ下線が付く
- S3 [単体]: Given ズレ0 When 描画 Then `±0:00` 表記でハッチ下線が付かない
  (境界値)。Given 計画0(%算出不能)When 描画 Then %は「-」(既存挙動を維持)
- S4 [単体]: Given `range=week` When 描画 Then 見出しが「今週のズレ」になる
- S5 [単体]: 符号付きH:MMフォーマッタ: 55分→`+0:55` / -70分→`-1:10` /
  0分→`±0:00`(境界値)
- S6 [単体]: Given LP When 描画 Then 見出し・サブコピー・
  「無料で始める」(→/signup)・「ログイン」(→/login)が表示される
- S7 [単体]: Given LP When 描画 Then 機能3件と使い方3ステップ、
  ベータ案内(Google連携は準備中)が表示される
- S8 [結合]: 既存のサマリー結合テスト(集計ロジック)と
  marketingレイアウトのフッターリンクテストが引き続き合格する(リグレッション)
