# 仕様書: P3-2 ギャップサマリー

- ステータス: 承認済み(2026-07-08)
- 関連: docs/要件定義書.md FR-07 / docs/指示資料(該当資料なし。README以外未配置)

## 目的

「今日/今週、計画に対して実績がどれだけズレたか」を数値で把握できるサマリー画面を追加する。カレンダーのオーバーレイ表示(P3-1)が視覚的なズレの把握なのに対し、本機能は集計値による定量的な把握を担う(要件定義書 §1 のコアバリュー)。

## 仕様

### 1. 画面

新規ルート `app/(app)/summary/page.tsx`(認証必須グループ。`app/(app)/layout.tsx`の認証チェックをそのまま利用)。

- タブ切り替え「今日」「今週」(`?range=today`|`?range=week`、デフォルト`today`。URLパラメータで状態保持、Server Componentのまま`<Link>`遷移で実装。カレンダーの日/週切り替えと同じ方針)
- カレンダー画面のヘッダーに「サマリー」への導線、サマリー画面に「カレンダーへ戻る」導線を追加する(現状カレンダー画面からの遷移手段がないため)
- 表示内容:
  1. 集計カード: 計画合計・実績合計・ズレ(時間・%)
  2. 予定単位の一覧: 予定名・計画時間・実績時間・ズレ。実績が0分の予定には「未着手」ラベル
  3. 割り込み・フリー作業一覧: 予定に紐づかない実績(フリータイマー、または紐づけ先の予定が対象期間内にない実績)を、タイトル・実績時間で一覧表示

### 2. 集計ロジック(新規: `lib/summary/aggregate.ts`)

純粋関数 `computeGapSummary(planEvents, actualEntries, range)` を新設する。

```ts
interface SummaryRange {
  start: Date;
  end: Date; // 半開区間 [start, end)
}
interface GapSummaryItem {
  googleEventId: string;
  title: string;
  planMinutes: number;
  actualMinutes: number;
  gapMinutes: number; // actual - plan(符号あり)
  notStarted: boolean; // actualMinutes === 0
}
interface InterruptionItem {
  id: string;
  title: string;
  actualMinutes: number;
}
interface GapSummary {
  planTotalMinutes: number;
  actualTotalMinutes: number;
  gapMinutes: number; // actualTotal - planTotal
  gapPercent: number | null; // planTotalMinutes===0のときnull。それ以外は(gapMinutes/planTotalMinutes)*100を四捨五入
  items: GapSummaryItem[]; // 開始時刻昇順
  interruptions: InterruptionItem[]; // 開始時刻昇順
}
```

集計対象の判定ルール:

- 予定・実績とも「開始時刻(`startAt`)が対象範囲`[start, end)`内にあるもの」を集計対象とする(範囲をまたぐ予定・実績は対象外。詳細はスコープ外を参照)
- 予定単位の一覧は範囲内の予定をすべて含む。各予定の`actualMinutes`は、同じ`googleEventId`を持ち範囲内に開始する実績エントリの時間を合算する(中断・再開で複数セグメントある場合も合算)
- 割り込み一覧は「`googleEventId`が`null`(フリータイマー)」または「`googleEventId`はあるが範囲内の予定に一致するものがない」実績を対象とする
- `planTotalMinutes`が0の場合、`gapPercent`は`null`とする(ゼロ除算を避ける)。画面には「-」を表示する

### 3. データ取得(`app/(app)/summary/page.tsx`)

既存の`fetchSyncedEvents` / `fetchTimeEntries` / `fetchRunningEntry`(いずれも基準日±1週間を返す)を`baseDate=現在時刻`で呼び出す。「今週」の範囲は基準日を含む週(月曜始まり)であり、既存の取得範囲(±1週間)に完全に含まれるため追加のDB呼び出しは不要。

実行中タイマーは`lib/timer/blocks.ts`の`actualBlockInputs`(P2-2で実装済み)で確定済み実績とマージし、開始〜現在時刻分を実績合計に含める。

### 4. 文言(新規: `lib/summary/messages.ts`)

日本語文言をここに集約する(既存の`lib/calendar/messages.ts`等と同じ方針)。

## スコープ外

- 対象期間をまたぐ予定・実績(前日から続く予定等)の按分計算。開始時刻基準で範囲に含まれるかどうかのみを判定する(P3-1のP2-1由来の日区切りロジックと同じ簡略化方針)
- 「今日」「今週」以外の期間指定(任意の日付・週の選択、月次集計等)
- 予定単位一覧・割り込み一覧のソート順変更、フィルタ、CSV等のエクスポート
- タスク種別ごとの見積もり係数分析(v2スコープ、CLAUDE.md参照)

## テストシナリオ

### 単体(`lib/summary/aggregate.ts`)

- S1 [単体]: Given 計画と実績が完全一致する予定が1件, When `computeGapSummary`を実行, Then `planTotalMinutes === actualTotalMinutes`、`gapMinutes: 0`、`gapPercent: 0`、対応する`item.notStarted: false`
- S2 [単体]: Given 実績が計画より多い(超過), When 集計, Then `gapMinutes`が正の値、`gapPercent`が正の値
- S3 [単体]: Given 実績が計画より少ない(早期終了), When 集計, Then `gapMinutes`が負の値、`gapPercent`が負の値
- S4 [単体・境界値]: Given 対象範囲内に予定が1件もない, When 集計, Then `planTotalMinutes: 0`、`gapPercent: null`(ゼロ除算にならない)
- S5 [単体・異常系]: Given 実績が紐づかない予定(未着手), When 集計, Then 対応する`item.notStarted: true`、`item.actualMinutes: 0`
- S6 [単体・異常系]: Given `googleEventId`が`null`の実績(フリータイマー), When 集計, Then `interruptions`に含まれる
- S7 [単体・異常系]: Given `googleEventId`は設定されているが対象範囲内の予定に一致するものがない実績, When 集計, Then `interruptions`に含まれる
- S8 [単体・境界値]: Given 対象範囲外に開始時刻を持つ予定・実績, When 集計, Then 集計対象・一覧のいずれからも除外される
- S9 [単体]: Given 同じ予定に紐づく実績が複数セグメントある(中断・再開), When 集計, Then 対応する`item.actualMinutes`はセグメントの合算値になる

### 結合(`app/(app)/summary/page.tsx`のレンダリング)

- S10 [結合]: Given 今日の範囲に予定・実績が存在する, When サマリーページ(`?range=today`)を表示, Then 計画合計・実績合計・ズレ(時間・%)が表示される
- S11 [結合]: Given 「今週」タブへ切り替える, When 表示, Then 週次の集計に切り替わる(`?range=week`遷移で値が変わることを確認)
- S12 [結合]: Given 未着手の予定がある, When 表示, Then 一覧に「未着手」ラベル付きで表示される
- S13 [結合]: Given 割り込み・フリー実績がある, When 表示, Then 割り込み一覧にタイトルと実績時間が表示される
- S14 [結合・境界値]: Given 対象範囲に予定が1件もない, When 表示, Then ズレ%は「-」で表示され、エラーにならない
- S15 [結合]: Given 実行中タイマーがある, When 今日のサマリーを表示, Then 実績合計に開始〜現在時刻分が加算される
