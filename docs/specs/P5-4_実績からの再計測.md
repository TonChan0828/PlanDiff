# 仕様書: P5-4 実績からの再計測(クリックで同名タイマー開始)

- ステータス: 承認済み(2026-07-19)
- 関連: docs/要件定義書.md の FR-04(タイマー)/ FR-11(計測画面)。`docs/指示資料/` に関連資料なし(2026-07-19確認)
- 発端: 2026-07-19 ユーザー要望「カレンダーの実績や過去に計測した記録などをクリックすることでその項目名称で計測をスタートできる機能」

## 目的

過去に計測した実績をワンアクションで「同じ名前・同じ予定紐づけ」の新規計測として再開できるようにする。
中断→再開や同じ作業の繰り返し計測のたびにフリータイマーへ名前を打ち直す手間をなくし、計測の継続率を上げる。

## 仕様

### 共通の開始動作

- 再計測は既存の `startTimerAction({ googleEventId, title })` を呼ぶだけで実現する。**DB変更・新規Server Actionなし**
- 入力は元実績のスナップショット:
  - `title`: 元実績のタイトルをそのまま使用(空文字なら空のまま=無題のフリー実績)
  - `googleEventId`: 元実績の値を**引き継ぐ**(予定紐づき実績の再計測は同じ予定の作業再開とみなす。フリー実績は `null` のまま)
- 開始時刻はサーバー側で「今」を決定(既存挙動)。実行中タイマーがあれば自動停止される(既存挙動。エラーにしない)
- 押下後は pending 状態にして二重開始を防ぐ。失敗時は楽観的更新をロールバックし、既存パターンの日本語エラーメッセージを表示する

### 導線1: 計測画面「今日の実績」リスト(`components/track-view.tsx`)

- 各項目の右側に再計測ボタン(▶アイコン)を追加する。紐づき実績・フリー実績の両方に表示
- フリー実績では既存の「予定にする」ボタンと並ぶ。375px幅で崩れないよう、再計測はアイコンボタン(44pxタップ領域)とする
- aria-label: 「◯◯を再計測」(無題の場合は既存の無題表記を使用)
- 押下で `startTimer(entry.googleEventId, entry.title)`(既存ヘルパー=楽観的更新+ロールバック+`router.refresh()`)を呼ぶ

### 導線2: カレンダーの実績ブロック(`components/calendar-view.tsx` の `ActualBlock`)

- `editable` な実績ブロックの**右上に▶再計測ボタンを重ねて表示**する。ブロック本体のタップ=編集パネルという既存動線は維持
- ブロック全体が1つの編集ボタンのため、ボタンの入れ子を避けて `li` 内の**兄弟要素**として絶対配置する
- **表示条件**: ブロックの描画高さが28px以上(`HOUR_PX=56` のため30分以上の実績に相当)。それ未満はボタンを出さない(誤タップ防止。コンテキストパネルが代替導線)
- 実行中ブロック(`editable: false`)には表示しない(既に計測中のため)
- aria-label: 「◯◯を再計測」。編集ボタンのaria-label(「◯◯の実績を編集」)と区別できること
- 押下でカレンダー側の楽観的更新パターン(既存 `handleStartTimer` / `handleStartFreeTimer` と同様)で開始する

### 導線3: コンテキストパネルの実績リスト(`components/calendar-context-panel.tsx`)

- 「実績」リストの各項目に、既存の鉛筆(編集)ボタンの隣に▶再計測ボタンを追加する
- `onRestartEntry: (entry: TimeEntryItem) => void` prop を新設し、`CalendarView` の再計測ハンドラに接続する
- aria-label: 「◯◯を再計測」

### 文言

- 新規UIテキストは既存辞書に追加する: `lib/timer/messages.ts`(カレンダー側)/ `lib/track/messages.ts`(計測画面側)/ `lib/calendar/messages.ts`(コンテキストパネル側、必要な場合)
- すべて日本語。ハードコードしない

## スコープ外

- 過去実績の検索・サジェスト(頻出タスクのランキング等)
- 元実績の開始/終了時刻・メモ等タイトル以外の情報の引き継ぎ
- 再計測時のタイトル編集ダイアログ(そのまま開始する。名前を変えたい場合は開始後に編集)
- サマリー画面への再計測導線

## テストシナリオ

正常系・異常系・境界値を含む。結合レベルは S8 を含む。

- S1 [単体]: Given 計測画面「今日の実績」に予定紐づき実績がある When 再計測ボタンを押す Then `startTimerAction` が元実績の `googleEventId` と `title` で呼ばれ、実行中表示に切り替わる
- S2 [単体]: Given 「今日の実績」にフリー実績(googleEventId=null)がある When 再計測ボタンを押す Then `googleEventId: null`・元タイトルで開始され、既存の「予定にする」ボタンも引き続き表示されている
- S3 [単体]: Given カレンダーに30分以上の確定済み実績ブロックがある When ブロック右上の▶ボタンを押す Then 元実績の `googleEventId`・`title` で開始され、編集パネルは開かない
- S4 [単体]: Given カレンダーに実行中の実績ブロック(editable=false)がある Then ▶ボタンは表示されない
- S5 [単体・境界値]: Given 30分(描画高さ28px)の実績ブロック Then ▶ボタンが表示される。Given 15分(14px)の実績ブロック Then ▶ボタンは表示されない
- S6 [単体]: Given コンテキストパネルの実績リストに実績がある When ▶ボタンを押す Then 元実績の値で開始され、鉛筆ボタンの編集動線は従来どおり動く
- S7 [単体・異常系]: Given 再計測ボタンを押して `startTimerAction` が失敗する Then 楽観的更新がロールバックされ、日本語のエラーメッセージが表示される
- S8 [結合]: Given 実行中タイマーが存在する状態 When 別の実績を再計測する Then 既存タイマーが自動停止され、元実績の `googleEventId`・`title` を引き継いだ新規実行中エントリが1件だけ作られる(`end_at IS NULL` が常に1本)
- S9 [単体・境界値]: Given タイトルが空文字の実績 When 再計測する Then 空タイトルのまま開始され、UIには無題表記が出る
- S10 [単体]: Given 再計測ボタン押下直後(pending中) When 同じボタンを連打する Then 開始処理は1回しか呼ばれない
