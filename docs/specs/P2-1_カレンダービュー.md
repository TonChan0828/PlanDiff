# 仕様書: P2-1 カレンダービュー

- ステータス: 承認済み(2026-07-07。既定=日ビュー、予定=左55%レーンで確定)
- 関連: docs/要件定義書.md の FR-03 / §6(カレンダー=メイン画面)/ §7.3 / §7.4、docs/開発計画.md P2-1
- 依存: P1-2(カレンダー同期。`synced_events` キャッシュと `POST /api/calendar/sync`)
- 指示資料: なし(docs/指示資料/ を確認済み、2026-07-07時点でREADMEのみ)

## 目的

P1-2の簡易リストを、時系列タイムライン(日/週ビュー)に置き換える。
予定ブロックを時間軸上に配置し、P2-2(タイマー)・P3-1(オーバーレイ)の土台となるレイアウトを確立する。
モバイルファースト(375px基準)。

## 仕様

### 全体構成

```
lib/calendar/view-date.ts     … 表示日付・週列・ナビゲーション先の計算(date-fns。クライアント/サーバー共用)
lib/calendar/layout.ts        … 予定ブロックの配置計算(時刻→位置%、日跨ぎクリップ、重複の横分割)
lib/calendar/messages.ts     … カレンダー画面のUI文言定数(既存文言もここへ集約)
components/calendar-view.tsx  … タイムライン本体(日/週切替、ナビゲーション、同期トリガ)※クライアント
app/(app)/calendar/page.tsx   … searchParams(view / date)を読み、該当期間のキャッシュを渡す
```

- `components/calendar-sync.tsx` の同期トリガ(マウント時+手動更新+401時の再認可誘導)は
  `calendar-view.tsx` に統合し、`calendar-sync.tsx` は削除する(簡易リスト表示の役目を終える)
- Google API・DBアクセスは従来どおりサーバーのみ。クライアントには `synced_events` 由来のデータだけを渡す

### URL状態(Server Componentsデフォルト維持のため)

- `/calendar?view=day|week&date=YYYY-MM-DD`
- 省略時: `view=day`、`date=今日`。不正値はデフォルトへフォールバック(エラーにしない)
- ナビゲーション(前/次、今日、ビュー切替)は `router.push`(searchParams更新)で行い、
  page(Server Component)が新しい期間のキャッシュを読み直す

### 画面レイアウト

**共通(日/週)**

- ヘッダー行: 「‹ 前へ」「今日」「次へ ›」ナビゲーション、日/週ビュー切替、「更新」ボタン(手動同期)。
  すべてタップターゲット44px以上
- 縦の時間軸: 0時〜24時、1時間ごとの目盛りとグリッド線。1時間=固定高さ(モバイルで1画面に約8時間が収まる程度)
- 初期スクロール位置: 8:00 付近(当日表示で現在時刻が8時以降なら現在時刻−1時間)
- 現在時刻ライン: 表示範囲に「今日」が含まれる場合のみ、現在時刻の位置に横線を表示(1分単位で配置。リアルタイム追従は不要、描画時点でよい)
- 日時はすべて端末のタイムゾーンで表示(DBはUTC。date-fns使用)

**予定ブロック(FR-06のオーバーレイ規約を先取り)**

- 日列の**左寄せ・薄い塗り**(列幅の約55%)。右側はP2-2以降の実績ブロック用に空けておく
- ブロック内: タイトル(なければ「(タイトルなし)」)+開始〜終了時刻。あふれる場合は省略(週ビューの狭い列は時刻を省きタイトルのみ)
- 高さ=所要時間に比例。15分未満の予定も最小高さ(タップ可能な高さ)を確保
- 日跨ぎ予定は日ごとにクリップして表示(23:00〜翌1:00 → 当日23:00〜24:00と翌日0:00〜1:00)
- 同時間帯に重複する予定は、予定レーン(左55%)内で横に等分割
- P2-1ではブロックはタップ不可の表示のみ(タイマー開始はP2-2)。ただしP2-2でボタン化する前提の構造にする

**日ビュー(モバイル既定)**

- 上部に週ストリップ(月〜日の7日分の日付チップ。タップでその日へ移動、当日・選択日を強調)
- 本体は選択日1日分のタイムライン

**週ビュー**

- 月曜始まりの7列。各列ヘッダーに曜日+日付(当日を強調)
- 375pxでは列が狭いため、ブロックはタイトルのみ(1〜2行で省略)。横スクロールは発生させない

### 同期との接続(P1-2の挙動を維持)

- キャッシュ表示を先行し、マウント時に `POST /api/calendar/sync` をバックグラウンド実行 → 成功時 `router.refresh()`
- 表示週が変わったら(ナビゲーションで週をまたいだら)、新しい「表示週±1週間」で同期を再実行する
  (期間計算は既存 `computeSyncRange` を基準日付き で再利用)
- 「更新」ボタンで手動同期。同期中はボタン無効化+「同期中…」
- 401 `reauthorize` → `/auth/reauthorize`、その他失敗 → 日本語エラー表示(キャッシュ表示は維持)
- 同一週内の日移動では再同期しない(キャッシュ済み期間内のため)

### データ変更

- なし(読み取りのみ。スキーマ・API変更なし)
- page の読み取り期間は「表示週±1週間」(P1-2と同じ。サーバーTZの概算で可、±1週バッファがTZ差を吸収)

### エラー時の挙動(メッセージはすべて日本語)

| ケース | 挙動 |
|---|---|
| 同期失敗(502等) | エラーメッセージ表示、キャッシュのタイムライン表示は維持 |
| 401 reauthorize | `/auth/reauthorize` へリダイレクト(P1-1導線) |
| 不正な searchParams | デフォルト(今日・日ビュー)にフォールバック |
| 予定が0件 | タイムライン(時間軸)は表示し、「予定がありません」を軸上に控えめに表示 |

### アクセシビリティ / UI品質(ui-quality Skill準拠)

- ナビゲーション・切替は `button` / `a` で実装。アイコンのみボタンには日本語 `aria-label`
- 色だけに意味を持たせない(予定ブロックは塗り+枠線+テキストで判別可能に)
- コントラスト比 4.5:1 以上。ライトテーマ基準で実装し、既存ページ同様 dark: も当てる
- 375px幅で横スクロールなし(完了前にPlaywrightで実機確認)

## スコープ外

- タイマー操作(予定タップで開始/停止)= P2-2
- 実績ブロックの表示・オーバーレイ = P2-2 / P3-1(本項目は予定レーンの位置取りまで)
- 終日イベント表示(P1-2で同期対象外と決定済み)
- スワイプによる日/週移動、ピンチズーム、時間軸の拡大縮小
- 月ビュー、複数週表示
- ビュー選択の永続化(localStorage等。URLのみで保持)

## テストシナリオ

### 単体(lib/calendar)

- S1 [単体] 正常系: Given 09:00〜10:30(ローカルTZ)の予定 When 配置計算 Then top=9/24・height=1.5/24 に相当する位置情報が返る
- S2 [単体] 境界値: Given 23:00〜翌01:00 の日跨ぎ予定 When 当日と翌日それぞれで配置計算 Then 当日は23:00〜24:00、翌日は00:00〜01:00 にクリップされる
- S3 [単体] 境界値: Given 同時間帯に重複する予定2件/3件 When 配置計算 Then レーン内で2等分/3等分の列に割り当てられる(重ならない)
- S4 [単体] 境界値: Given 5分の極短予定・開始=終了の予定 When 配置計算 Then 最小高さが確保され、高さ0にならない
- S5 [単体] 正常系: Given 任意の日付 When 週列を計算 Then 月曜始まりの7日が返り、前へ/次へのナビゲーション先(日ビュー=±1日、週ビュー=±7日)が正しい
- S6 [単体] 異常系: Given 不正な `date`/`view` パラメータ When 解釈 Then 今日・日ビューにフォールバックする

### コンポーネント(components/calendar-view)

- S7 [単体] 正常系: Given 予定2件のキャッシュ When 日ビューを描画 Then 時間軸(0〜24時)上にタイトル・時刻付きの予定ブロックが時系列で表示される
- S8 [単体] 正常系: Given 週ビュー指定 When 描画 Then 月曜始まり7列に予定ブロックが配置され、当日列が強調される
- S9 [単体] 正常系: Given 日ビュー表示中 When 「次へ」をタップ Then `date` が+1日のURLへ遷移する(週ビューなら+7日)
- S10 [単体] 正常系: Given 表示週が変わるナビゲーション When 発火 Then 新しい表示週±1週間の期間で同期POSTが送られる。同一週内の日移動では送られない
- S11 [単体] 正常系: Given 予定0件 When 描画 Then 時間軸と「予定がありません」が表示される
- S12 [単体] 異常系: Given 同期APIが502 When マウント同期 Then 日本語エラーメッセージが表示され、キャッシュのブロック表示は維持される(401 reauthorize時は /auth/reauthorize へ)
- S13 [単体] 正常系: Given 今日を含む表示 When 描画 Then 現在時刻ラインが表示される(今日を含まない週・日では表示されない)

### 結合(page 〜 ローカルDB)

- S14 [結合] 正常系: Given `synced_events` に表示週内・週外(±1週内)・範囲外の行がある When `?date=` 指定でpageのデータ取得を実行 Then 表示週±1週間の行だけが取得され、範囲外は含まれない
- S15 [結合] 境界値(分離): Given 別ユーザーの行が同期間にある When ユーザーAで取得 Then 別ユーザーの行は含まれない(RLS)

### 手動確認(DoDに含める)

- M1: 実Googleアカウントの予定が、日/週ビューの正しい時刻位置に表示される(375pxで横スクロールなし)
- M2: 週ナビゲーションで前週・翌週に移動すると、その週の予定が同期されて表示される
