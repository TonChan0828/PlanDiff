# 仕様書: D-5 ロゴ作成

- ステータス: 承認済み(2026-07-19)
- 関連: docs/specs/P3-3_PWA対応.md(暫定アイコンの置き換え)/ docs/specs/D-1-2_デザイン刷新.md(ブランドカラー)/ docs/指示資料/(2026-07-19時点でロゴ関連の資料なし)

## 目的

PlanDiffの正式なロゴマークを作成する。現状のPWAアイコンはP3-3で「デザイン素材が未用意のため」の暫定品(群青背景+白文字「P」)であり、faviconもNext.jsデフォルト由来のまま。プロダクトの「顔」である予定/実績オーバーレイをモチーフにしたロゴマークをSVGで設計し、アプリ全体(PWAアイコン・favicon・apple-icon・ヘッダー)へ適用してブランドの一貫性を確立する。

## 仕様

### モチーフ(ユーザー確認済み 2026-07-19)

予定/実績オーバーレイの象徴化。**左上に「予定」ブロック(薄い群青)、右下に「実績」ブロック(濃い群青)がズレて重なる**構成で、「計画と現実のギャップ」を表現する。

- 色はブランドトークンに準拠: 群青 `#2f4acb`(ライト)/ `#4c6ef5`(ダーク)、必要に応じて柿 `#d9480f` をアクセントに使用可
- バリエーション3案(A: 斜めオフセットの角丸ブロック2つ / B: ガント風横バー2本 / C: アウトライン枠+はみ出す塗りブロック)をSVGで作成し、プレビュー(実サイズ・ライト/ダーク・maskableセーフゾーン付き)でユーザーが1案選定してから適用する
- maskableアイコンの安全領域(中央80%)にモチーフを収める(P3-3の方針を維持)

### 適用範囲(入出力・画面)

| 対象 | 変更 |
|---|---|
| `components/logo-mark.tsx`(新規) | 選定SVGのReactコンポーネント。named export、`className` でサイズ指定、`aria-hidden="true"`(装飾。名称は隣接するワードマーク/aria-labelが担う) |
| `components/wordmark.tsx` | `withMark?: boolean` プロパティを追加。`true` でロゴマークをテキストの前に表示。未指定時は現状表示を維持 |
| `components/app-bar.tsx` / `components/marketing-header.tsx` / `components/desktop-nav.tsx` | `<Wordmark withMark />` に変更し、全ヘッダーでロゴマーク+ワードマークを表示 |
| `lib/pwa/app-icon.tsx` | `generateAppIcon()` の描画を白「P」から選定モチーフに刷新(群青背景は維持、ブロックはその上に白/半透明白で描画。`ImageResponse` 内のdiv/svgで実装) |
| `app/icon.svg`(新規) | Next.jsファイル規約によるSVG favicon。`prefers-color-scheme` でダーク対応 |
| `app/favicon.ico`(削除) | icon.svgに置き換え |
| `app/icon-192/route.tsx` / `app/icon-512/route.tsx` / `app/apple-icon.tsx` / `app/manifest.ts` | 変更なし(`generateAppIcon` 経由で新デザインが自動反映) |

### エラー時の挙動

ランタイム入力を持たない静的アセット/純粋表示コンポーネントのため、ユーザー操作起因のエラーは発生しない。ビルド時生成(`force-static`)が失敗した場合はビルドエラーとして検出される(`npm run check` で担保)。

## スコープ外

- OGP画像(SNSシェア用)— 必要になったら別タスク化
- モノクロ版・単色版などロゴバリアントの整備、ブランドガイドライン文書
- スプラッシュスクリーン等のPWA追加アセット

## テストシナリオ

結合レベルは「アイコンRoute Handler(ビルド時生成)〜PNG出力」と「manifest宣言との整合」を対象とする。

- S1 [単体]: Given `LogoMark` コンポーネント When レンダリングする Then `svg` 要素が `viewBox` と `aria-hidden="true"` を持ち、予定/実績の2ブロック(rect要素2つ以上)を含む
- S2 [単体]: Given `LogoMark` に `className` を渡す When レンダリングする Then ルート `svg` にそのクラスが適用される
- S3 [単体]: Given `Wordmark` を `withMark` なしで使う When レンダリングする Then ロゴマークは表示されず現状の「Plan」+「Diff」表示のまま(後方互換)
- S4 [単体]: Given `Wordmark` を `withMark` 付きで使う When レンダリングする Then ロゴマーク(svg)とテキストが両方表示され、`aria-label="PlanDiff"` が維持される
- S5 [単体]: Given `AppBar` / `MarketingHeader` / `DesktopNav` When レンダリングする Then ヘッダー内にロゴマーク(svg)が表示される
- S6 [結合]: Given `/icon-192` `/icon-512` のRoute Handler When GETする Then 200・`image/png` で、実PNGサイズが192x192 / 512x512(manifestの宣言と一致)
- S7 [結合]: Given `apple-icon` When 生成する Then 200・`image/png` で実PNGサイズが180x180
- S8 [単体・境界値]: Given `generateAppIcon(192)`(最小適用サイズ) When 生成する Then エラーなくPNGが生成される(モチーフ描画がサイズ非依存であること)
- S9 [単体]: Given `app/icon.svg` When ファイルを読み込む Then SVGとして妥当(`<svg` と `viewBox` を含む)で、ブランド群青 `#2f4acb` を含む。かつ `app/favicon.ico` が存在しない
