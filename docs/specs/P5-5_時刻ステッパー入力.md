## 目的

実績・実行中タイマー・アプリ内予定の時刻編集は、現在ネイティブの
`<input type="datetime-local">` で実装されている。ブラウザ標準のこの入力は
分セグメントを `00` から下方向に操作すると `59` に折り返すだけで、**時セグメントには
波及しない**(セグメントが独立して動く)。ユーザーからの要望は「分を減らして
`00分` → `59分` になるとき、時刻(時)も `-1` してほしい」という、実際の時刻演算として
自然な繰り下がり/繰り上がり挙動。ネイティブ入力のセグメント挙動は上書きできないため、
**独自の時刻ステッパー(日付 + 時[▲▼] : 分[▲▼])に置き換える**。

## 仕様

### 1. 純関数: `lib/ui/local-date-time.ts`(新規)

ローカル日時文字列(`"yyyy-MM-dd'T'HH:mm"`。既存の `DATETIME_LOCAL_FORMAT` と同一形式)
に対する桁跨ぎ計算を担う。**Date の直接演算はせず**、date-fns の `parse` / `addMinutes` /
`addHours` / `format` を組み合わせて実装する(月末・年跨ぎ・うるう年を date-fns に委譲)。

```ts
export const LOCAL_DATE_TIME_FORMAT = "yyyy-MM-dd'T'HH:mm";

export interface LocalDateTimeParts {
  date: string;   // "yyyy-MM-dd"
  hour: number;   // 0-23
  minute: number; // 0-59
}

export function parseLocalDateTime(value: string): LocalDateTimeParts | null;
export function formatLocalDateTime(parts: LocalDateTimeParts): string;

// 分を±1。00分で-1なら時も-1(日付跨ぎも連動)。59分で+1なら時も+1
export function stepLocalMinute(value: string, delta: 1 | -1): string;

// 時を±1。00時で-1なら日付も-1。23時で+1なら日付も+1
export function stepLocalHour(value: string, delta: 1 | -1): string;

// 直接入力の確定用。範囲外はクランプ(0-23 / 0-59)。桁は跨がない
export function setLocalHour(value: string, hour: number): string;
export function setLocalMinute(value: string, minute: number): string;

// 日付部分のみ置換。date="" なら value 全体を "" にする(未入力扱い)
export function setLocalDate(value: string, date: string): string;
```

- `value` が空文字列または不正な形式の場合、`step*`/`set*` 系はすべて `value` を
  そのまま返す(副作用なし)。`parseLocalDateTime` は `null` を返す
- DST(夏時間)は対象外とする(スコープ外に明記)。ローカル文字列上の単純な桁演算であり、
  実際のタイムゾーン切り替えは考慮しない(主対象は日本 = JSTで DST なし)

### 2. 共有コンポーネント: `components/date-time-stepper.tsx`(新規)

```ts
export interface DateTimeStepperProps {
  label: string;                    // 例: 「開始時刻」。legend と各パーツの aria-label 接頭辞
  value: string;                    // "yyyy-MM-dd'T'HH:mm"。未入力は ""
  onChange: (next: string) => void;
  disabled?: boolean;
}

export function DateTimeStepper(props: DateTimeStepperProps): JSX.Element;
```

- 構造: `<fieldset disabled={disabled}><legend>{label}</legend>` の中に
  「日付入力」「時ステッパー」「分ステッパー」を横並びで配置
- **日付**: ネイティブ `<input type="date">`(`aria-label="${label}の日付"`)。
  クリアすると `onChange("")` を呼ぶ(時分もリセットされ未入力状態になる)
- **時・分**: `role="spinbutton"` の `<input inputMode="numeric">` + 上下ボタン
  - `aria-valuenow` / `aria-valuemin=0` / `aria-valuemax={23|59}` /
    `aria-label="${label}の時"` または `"${label}の分"`
  - 常に2桁ゼロ埋め表示、`tabular-nums` で桁の暴れを抑える
  - 上ボタン(`aria-label="${label}の{時|分}を1進める"`)/
    下ボタン(`aria-label="${label}の{時|分}を1戻す"`)は `stepLocalHour` /
    `stepLocalMinute` を呼ぶ(**桁跨ぎあり**)。タップ領域 `min-h-11 min-w-11`
  - フォーカス中に `ArrowUp` / `ArrowDown` キーを押すと `preventDefault` した上で
    ボタンと同じ桁跨ぎ処理を実行する
  - 直接入力: フォーカス中はドラフト文字列を保持し、`blur` または `Enter` で確定。
    数値化できれば `setLocalHour` / `setLocalMinute` でクランプして確定(**桁跨ぎしない**)。
    数値化できなければ確定前の値に復元する
- `value === ""` のとき時・分の入力とボタンは `disabled`。直近の有効な時分を
  コンポーネント内で `useRef` に保持し、日付が再入力されたときに復元する
  (保持されていなければ `00:00` を初期値にする)
- 文言は `lib/ui/messages.ts`(新規)に集約する(既存 `lib/timer/messages.ts` 等と
  同じ「メッセージオブジェクトを1箇所に寄せる」規約に合わせる)

### 3. 適用箇所(3パネル。state・保存処理は変更しない)

いずれも既存の state 型(`"yyyy-MM-dd'T'HH:mm"` のローカル文字列)と
`handleSubmit` 内の `new Date(local).toISOString()` 変換をそのまま維持し、
`<input type="datetime-local">` のブロックを `<DateTimeStepper>` に差し替えるのみ。

| パネル | 対象フィールド |
|---|---|
| `components/edit-start-panel.tsx`(D-4) | 開始時刻(実行中タイマー) |
| `components/edit-entry-panel.tsx`(P2-4) | 開始時刻・終了時刻(確定済み実績) |
| `components/app-event-panel.tsx`(P2-5 / P5-1) | 開始時刻・終了時刻(アプリ内予定・定期予定) |

`components/recurring-rule-panel.tsx` の `type="time"`(時刻のみ・日付を持たない入力)は
**対象外**(スコープ外)。

## スコープ外

- `components/recurring-rule-panel.tsx` の時刻のみ入力(`type="time"`)の置き換え
- DST(夏時間)を考慮した桁跨ぎ(日本時間のみを主対象とするため非対応)
- DBスキーマ・RLSの変更(なし。表示・入力コンポーネントのみの変更)
- 秒単位の入力(既存どおり分単位)

## テストシナリオ

### 純関数(`lib/ui/local-date-time.ts`)

- S1 [単体]: Given `"2026-07-20T10:30"` When `stepLocalMinute(value, -1)` Then
  `"2026-07-20T10:29"`(通常の−1、境界値でない)
- S2 [単体]: Given `"2026-07-20T10:00"` When `stepLocalMinute(value, -1)` Then
  `"2026-07-20T09:59"`(00分で−1 → 時も−1。本要望の中心シナリオ)
- S3 [単体]: Given `"2026-07-20T10:59"` When `stepLocalMinute(value, 1)` Then
  `"2026-07-20T11:00"`(59分で+1 → 時も+1)
- S4 [単体]: Given `"2026-07-08T00:00"` When `stepLocalMinute(value, -1)` Then
  `"2026-07-07T23:59"`(日付跨ぎ)
- S5 [単体]: Given `"2026-08-01T00:00"` When `stepLocalMinute(value, -1)` Then
  `"2026-07-31T23:59"`(月末跨ぎ、境界値)
- S6 [単体]: Given `"2027-01-01T00:00"` When `stepLocalMinute(value, -1)` Then
  `"2026-12-31T23:59"`(年跨ぎ、境界値)
- S7 [単体]: Given `"2028-03-01T00:00"` When `stepLocalMinute(value, -1)` Then
  `"2028-02-29T23:59"`(うるう年、境界値)
- S8 [単体]: Given `"2026-07-20T00:00"` When `stepLocalHour(value, -1)` Then
  `"2026-07-19T23:00"`(時の繰り下がりで日付も−1)
- S9 [単体]: Given `"2026-07-20T23:00"` When `stepLocalHour(value, 1)` Then
  `"2026-07-21T00:00"`(時の繰り上がりで日付も+1)
- S10 [単体]: Given `"2026-07-20T10:30"` When `setLocalHour(value, 99)` Then
  `"2026-07-20T23:30"`(上限クランプ、桁は跨がない=分は変化しない)
- S11 [単体]: Given `"2026-07-20T10:30"` When `setLocalMinute(value, -5)` Then
  `"2026-07-20T10:00"`(下限クランプ)
- S12 [単体]: Given `""` When `stepLocalMinute("", -1)` / `stepLocalHour("", 1)` /
  `setLocalHour("", 5)` Then いずれも `""` を返す(異常系・未入力)
- S13 [単体]: Given `"invalid"` When 各関数を呼ぶ Then 元の文字列をそのまま返す
  (異常系・不正形式)。`parseLocalDateTime("invalid")` は `null`
- S14 [単体]: Given `"2026-07-20T10:30"` When `setLocalDate(value, "")` Then `""`
  (日付クリアで全体が未入力になる)

### `DateTimeStepper` コンポーネント

- S15 [単体]: Given `value="2026-07-20T10:00"` When 分の下ボタンをクリック Then
  `onChange("2026-07-20T09:59")` が呼ばれる(00分→時−1の実UI疎通)
- S16 [単体]: Given `value="2026-07-20T10:59"` When 分の上ボタンをクリック Then
  `onChange("2026-07-20T11:00")` が呼ばれる
- S17 [単体]: Given 分の入力に フォーカス When `ArrowDown` キーを押す Then
  クリックと同じ桁跨ぎ処理で `onChange` が呼ばれる
- S18 [単体]: Given 時の入力 When 直接 `"7"` と入力して blur Then
  `onChange("...T07:mm")`(ゼロ埋め・クランプされて確定。桁は跨がない)
- S19 [単体]: Given 時の入力 When 直接 `"abc"` と入力して blur Then
  `onChange` は呼ばれず表示が元の値に復元する(異常系)
- S20 [単体]: Given `value=""` When 描画 Then 時・分の入力とボタンが `disabled`
  になっている(境界値・未入力)
- S21 [単体]: Given `value=""` When 日付を入力 Then `onChange` が
  `"<入力日付>T00:00"`(直近の有効値がない場合のデフォルト)で呼ばれる
- S22 [単体]: Given 日付入力 When クリアする Then `onChange("")` が呼ばれる
- S23 [単体]: `getByRole("spinbutton", { name: "開始時刻の時" })` 等でラベル付き
  spinbutton が取得できる(a11y確認)

### パネル組み込み(既存パネルのテスト改修で担保。結合)

- S24 [結合]: Given `edit-start-panel` を開く When 分ステッパーで00分から
  1つ下げる Then フォームの内部状態が時−1・分59になり、保存すると
  `onSave` に渡る ISO 文字列がその時刻を反映している
- S25 [結合]: Given `edit-entry-panel` を開く When 開始・終了それぞれの
  ステッパーを操作して保存 Then `onSave` に渡る `startAt`/`endAt` が
  UTC ISO として正しく変換されている(既存の変換ロジックとの結合)
- S26 [結合]: Given `app-event-panel`(定期予定作成モード)で時刻を
  ステッパーで変更 When 保存 Then `onSaveRecurring` に渡る `startTime`/`endTime`
  (`HH:mm` 文字列)が変更後の値になっている(既存の繰り返し予定分岐との結合)
