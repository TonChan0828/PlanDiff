# 仕様書: デザイン刷新「時間のdiff」(D-1 基盤とアプリシェル / D-2 オーバーレイ磨き込み)

- ステータス: 承認済み(2026-07-12)
- 関連: docs/要件定義書.md の FR-03(カレンダービュー)・FR-06(オーバーレイ表示)/
  docs/開発計画.md の P4-5(磨き込み)の前倒しに相当 /
  デザインモック: https://claude.ai/code/artifact/64ba1855-fcee-441f-93f1-f79dfc468eef
  (2026-07-12 ユーザー確認済み)/ docs/指示資料: 該当なし(2026-07-12時点)
- 前提: **機能・データ・APIは一切変更しない**。見た目(トークン・タイポグラフィ・
  レイアウトシェル・オーバーレイの造形)のみを対象とするリファクタリング

## 目的

現状のUIは素のTailwind既定値(zinc枠線+skyブルー+ピル型ボタン)の組み合わせで、
プロダクトの記憶に残る要素がない。また `globals.css` の `font-family: Arial` が
ロード済みのGeistフォントを上書きするバグがあり、ナビゲーションは画面ごとに
手作りで構成が不一致。これらを「時間のdiff」コンセプト(計画と現実の差分を、
エンジニアが毎日読むdiffの文法で見せる)のトークン体系で一新し、プロダクトの
「顔」であるオーバーレイに最大の面積と品質を与える。

## 仕様

### D-1a. デザイントークン(globals.css の @theme に一元化)

| トークン(名) | Light | Dark | 用途 |
|---|---|---|---|
| 方眼紙 / 宵(背景) | `#F6F7F9` | `#0E1116` | ページ背景 |
| 面(サーフェス) | `#FFFFFF` | `#151A23` | カード・タイムライン背景 |
| 墨(テキスト) | `#1A1D23` | `#E8EAEF` | 基本テキスト |
| 墨・淡(補助) | `#5A6272` | `#9AA3B2` | 補助テキスト |
| 枠線 | `#E3E6EB` | `#262C38` | ボーダー |
| 群青(ブランド・実績) | `#2F4ACB` | `#4C6EF5` | 実績ブロック・アクティブタブ・主ボタン |
| 群青10%(計画) | `rgba(47,74,203,.10)`+枠`38%` | `rgba(92,124,250,.15)`+枠`45%` | 予定ブロック |
| 柿(割り込み・ズレ) | `#D9480F` | `#E8590C` | フリー/割り込み実績・ズレのハッチ |
| 紅(危険・現在) | `#DC2626` | `#F87171` | 停止ボタン・削除・現在時刻線・エラー |

- 色はコンポーネントへの直書き(`sky-600` 等)をやめ、上記トークン経由で参照する
- `lib/pwa/theme.ts` の `BRAND_COLOR` を `#2F4ACB` に、`DARK_BACKGROUND_COLOR` を
  `#0E1116` に変更する(manifest / theme-color / アプリアイコン各routeが追随)
- ライト/ダークは現行どおり `prefers-color-scheme` で切り替える

### D-1b. タイポグラフィ(Arialバグの修正)

- `globals.css` の `body { font-family: Arial, ... }` を削除し、
  `--font-geist-sans` + 日本語フォールバック(`"Hiragino Sans", "Noto Sans JP",
  sans-serif`)を適用する
- **時刻・経過時間・分数・ズレ表記・日付数字はすべて Geist Mono**(`--font-geist-mono`)
  + `tabular-nums` で描画する(時間の書体を等幅に統一)
- 新しい書体は追加ロードしない(既存のGeist 2書体のみ)

### D-1c. アプリシェル((app) 共通レイアウト)

- `app/(app)/layout.tsx` に共通シェルを実装する:
  - **AppBar**(上部・1行): 左にワードマーク(`Plan`=群青アウトライン文字+
    `Diff`=群青塗り。`-webkit-text-stroke` 非対応環境は淡色フォールバック)、
    右にページ固有アクション(children スロット)
  - **BottomTabBar**(下部固定): カレンダー `/calendar`・計測 `/track`・
    サマリー `/summary`・設定 `/settings` の4タブ。現在のページに
    `aria-current="page"` を付け群青で強調。アイコン+ラベル、各タブ44px以上、
    `safe-area-inset-bottom` を考慮
- 各ページのピル型ナビ(calendar: 計測/サマリー/設定/ログアウト、track: 3リンク、
  summary: カレンダーへ戻る、settings: カレンダーへ戻る)を**すべて削除**し、
  タブに集約する
- カレンダーページの「◯◯さんとしてログイン中です。」表示を削除する
  (メールアドレスは設定画面で確認できる。ログアウトも設定画面に既存)
- ページ見出し(`h1`)は維持する(サマリー・計測・設定)。カレンダーは
  ワードマーク+ツールバーが見出しを兼ねるため `h1` テキストは視覚非表示
  (`sr-only`)とする

### D-1d. カレンダーのツールバーとタイムライン面積

- ツールバー(1行): `‹` `今日` `›` +期間ラベル(等幅数字)+`日|週`トグル+
  `予定を追加`(群青の主ボタン)。Google連携が有効な場合のみ`更新`を表示
  (凍結フラグの挙動は現状維持)
  ※モックではアプリバーに日付ナビを置いたが、「今日」「日|週」の機能維持を
  優先しツールバー行に集約する
- タイムラインの `max-h-[65dvh]` を廃止し、**ビューポートの残り高さまで拡大**する
  (下部タブ・実行中バーと重ならない。内部スクロール・初期スクロール位置の
  挙動は現状維持)
- ボタン類の形状は `rounded-full` ピルから半径10〜14pxの矩形系に統一する

### D-1e. テーマの手動切替(2026-07-12 M1後にユーザー要望で追加)

- 設定画面に「外観」セクションを追加し、**ライト / ダーク / システムに合わせる**の
  3択(ラジオ)で切り替えられるようにする。初期値は「システムに合わせる」
- 選択は `localStorage`(キー `plandiff-theme`)に保存し、次回以降も維持する
- 実現方式: `<html>` の `data-theme` 属性で上書きする
  - `data-theme="dark"` → ダークトークンを適用(OS設定に関わらず)
  - `data-theme="light"` → ライトトークンを適用(OSがダークでも)
  - 属性なし(=システム)→ 従来どおり `prefers-color-scheme` に追随
  - `color-scheme` も属性に合わせて `dark` / `light` に固定する
- 初回描画のちらつき(FOUC)防止のため、ルートレイアウトの `<body>` 先頭に
  同期インラインスクリプトを置き、描画前に `localStorage` の値を反映する。
  不正値・読み取り失敗(プライベートモード等)は無視してシステム追随とする
- 対象は全ページ(marketing含む。トークンはルートで共有のため)

### D-2. オーバーレイの磨き込み(プロダクトの「顔」)

- **配色の置き換え**(挙動・レーン構成・55/45比率は現状維持):
  - 予定ブロック: sky系 → 群青10%の薄い塗り+群青38%の枠
  - 実績ブロック(紐づき): sky-600 → 群青の濃い塗り+白文字
  - フリー/割り込み実績: amber系 → 柿の濃い塗り+白文字
  - ズレのハッチ(`gap-delay` / `gap-overrun`): amber → 柿の斜線
    (`repeating-linear-gradient` 135deg・模様による冗長表現は維持)
- **ズレ表記を符号付きdiff形式に変更**(`lib/timer/messages.ts`):
  - `delayLabel`: 「開始17分遅れ」→「**+17分 遅れ**」
  - `overrunLabel`: 「25分超過」→「**+25分 超過**」
  - 数値部分は等幅。aria-label・サマリー等、同関数の参照箇所すべてに波及する
- **方眼背景**: タイムラインに15分刻みの薄い水平線+同程度の薄い垂直線を敷く
  (1時間線はやや濃く)。時間軸ラベルは等幅
- **現在時刻線**: 紅の水平線に現在時刻チップ(等幅・紅地白文字)を追加する
- **実行中表現**: 実績レーンの記録中ブロックに白点滅ドット+「記録中」。
  点滅は `prefers-reduced-motion: reduce` で停止する
- **実行中バー / フリータイマーバー**: 面+枠線のカード(半径14px)に変更。
  経過時間は等幅・柿のドット(記録中)・停止ボタンは紅。sticky配置は
  タブバー直上に変更
- 週ストリップ・週ビューヘッダー: 選択日=群青の塗り、今日=群青の枠に統一
- 計測画面のクイックスタート・当日実績カード、サマリーのカード類も
  同トークンに追随させる(**レイアウト変更はしない**。色・書体・半径のみ)

## テストシナリオ

- S1 [単体]: Given 認証済みシェル When `/calendar` でBottomTabBarを描画
  Then カレンダー/計測/サマリー/設定の4リンクが表示され、カレンダーのみ
  `aria-current="page"` を持つ
- S2 [単体]: Given `?view=week&date=2026-07-06` 付きの `/calendar` When 描画
  Then カレンダータブがアクティブになる(境界値: クエリ付きURL)
- S3 [単体]: Given タブ対象外のパス(`/onboarding`)When 描画
  Then どのタブも `aria-current` を持たない(異常系)
- S4 [単体]: Given `delayLabel(17)` / `overrunLabel(25)` / `delayLabel(1)`
  When 呼び出す Then 「+17分 遅れ」「+25分 超過」「+1分 遅れ」を返す
  (境界値: 1分)
- S5 [単体]: Given カレンダーページ When 描画 Then ログアウトボタン・
  計測/サマリー/設定のピルリンク・「さんとしてログイン中」を含まない
  (ナビ集約の確認)
- S6 [単体]: Given AppBar When ページ固有アクションを渡して描画
  Then ワードマーク(PlanDiff)とアクションが表示される
- S7 [単体]: Given 予定に紐づく実績が開始遅延+超過 When オーバーレイ描画
  Then ハッチ(`gap-delay` / `gap-overrun`)と符号付きラベルが表示される
  (既存オーバーレイテストの表記追随を含む)
- S8 [結合]: Given 認証済みユーザー(Supabaseクライアントはモック)
  When (app)レイアウトを子ページとともに描画 Then 共通のタブナビ・
  ワードマーク・子コンテンツがすべて含まれる(レイアウト↔シェルの
  コンポーネント間連携。DBを跨ぐ流れは既存結合テストS9で担保)
- S9 [結合]: 既存結合テスト全件(タイマー・予定CRUD・サマリー集計・認証)が
  新UI構造でも合格する(リグレッション。既存64件を維持)
- S10 [単体]: Given 保存値なし When 外観セレクタを描画
  Then ライト/ダーク/システムの3択が表示され「システムに合わせる」が選択済み
- S11 [単体]: Given 外観セレクタ When ダーク(またはライト)を選択
  Then `<html data-theme>` が該当値になり、localStorageに保存される
- S12 [単体]: Given ダーク選択済み When 「システムに合わせる」を選択
  Then `data-theme` 属性が除去され、保存値が `system` になる(復帰の確認)
- S13 [単体]: Given localStorageの保存値(dark / light / 不正値 / 取得失敗)
  When 初期化スクリプト・解決関数を評価 Then dark/lightのみ属性に反映され、
  不正値・失敗時は属性なし(システム追随)になる(異常系・境界値)
- S14 [結合]: Given 設定ページ When 描画 Then 外観セクション(3択)が
  アカウント等の既存セクションと同居して表示される(ページ↔セレクタの連携)
