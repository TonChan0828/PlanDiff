## 07:00 P5-2 CI失敗の修正(タイムゾーン依存バグ)

- やったこと:
  - PR #28のCI(GitHub Actions=UTC実行)で `suggestions.test.ts` が12件失敗。
    `TZ=UTC npx vitest run` でローカル再現し原因を特定
  - 原因: `lib/calendar/suggestions.ts` の `addDaysTz` が `new TZDate(date)`(コピーコンストラクタ)
    を使っており、**TZDateのコピーコンストラクタはタイムゾーンを引き継がず実行環境のTZになる**。
    JSTのローカルでは偶然一致して合格し、UTCのCIで週境界・提案日の計算がずれた
  - 修正: `addDaysTz(date, days, timeZone)` にTZを明示して `new TZDate(date.getTime(), timeZone)`
    で複製するよう変更(P5-1のrecurring.tsが常にTZ明示で生成しているのと同じ流儀に統一)
  - `TZ=UTC` / `TZ=America/New_York` / ローカル(JST)の3タイムゾーンで単体31件の合格を確認、
    `npm run check` 合格
- 理由: CIと開発機のTZ差で挙動が変わる実装はタイムゾーン処理の欠陥であるため
  (テストがUTCで落ちたのは、S11等のTZシナリオが機能した正しい検出)
- 結果: CI再実行で合格見込み(push済み)。TZDateのコピーコンストラクタは要注意(発生1回目)
- 未解決・次にやること: CIグリーン確認 → ユーザーレビュー・マージ待ち
